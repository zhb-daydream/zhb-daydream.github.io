创建本地git密钥

发表于2024-04-26|更新于2024-04-26|博客搭建
|字数总计:543|阅读时长:1分钟

写在前面

因为换了新电脑，连带着本地的 git 密钥也得再弄一遍，才能在新电脑上完成 Hexo deploy。虽然之前已经补了一个博客去记录怎么把 Hexo 网页部署到 Github 上了，但是“在本地通过 git 生成密钥”这个步骤是省略掉了的，而这个 Github 入门级别的操作我在初学那会也没记录下来，所以还是补一下吧！

一般来说，本地的 git 与远程的 github/gitlab/gitee 仓库之间的数据交换是通过 SSH 加密传输的，很少会使用到 https 的连接方式，因为后者需要在每次 pull 和 push 的时候输入用户名和密码来验证登录，操作繁琐。而若是需要通过 SSH 加密且“免密”传输，就需要提前在两端设置好对应的密钥-公钥，这样传输的时候直接比较密钥-公钥是否匹配，就能完成数据传输通道的建立了。所以生成并配置 ssh 密钥是使用 git 管理代码的基本操作。
