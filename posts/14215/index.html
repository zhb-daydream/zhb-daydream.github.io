Hexo清空git的deploy仓库

发表于2024-04-06|更新于2024-04-19|博客搭建
|字数总计:880|阅读时长:3分钟

起因

昨晚大半夜照着百度搜索引擎的 AI 踉踉跄跄把批量导入相册路径的 python 脚本写好了，hexo g & hexo s 的本地显示也是没问题的，结果 deploy 之后就说相册页面没找到文件。昨晚可能也是因为晚上八点的时候刚喝了个咖啡，心跳也比较快，感觉得赶紧睡了，不然肯定得猝死，只能抱憾睡觉，脑子里念叨着这事好久才睡着。

起床之后先看了下 hexo g 在本地生成的 public 仓库，里面的 html 也正常。重新翻了翻其他 up 主关于创建相册的教程，看起来也没什么问题（不过大都是直接照搬官方文档的说明，有一个是带着写 .md 文件，但也没有把 front matter 加进去）。

最后才把问题排查方向定位回 “本地显示网页正常，远程网页显示不正常” 上面。去 deploy 的仓库里面一看，好家伙，存储相册的文件夹是大写的 Gallery 而不是我使用的 gallery，由于 _config.yml、图片路径全都是用的小写版本的 gallery，所以远程部署的网页都没有找到相应文件，就报错了。

那么这个大写的 Gallery 是怎么来的呢？我明明已经 hexo clean 了，为什么还会有的呢？这个大写文件夹实际上是我最开始用的版本，后来想和 tags、categories 统一才改成小写，很可能是大写的 Gallery 文件夹存进了某种缓存里面，后续 deploy 的时候又没有清空掉，所以就一直留着了。问题排查到这里，才算是把真正的问题定位到 “Hexo 在 deploy 之前如何清除掉现有缓存”。

清除git缓存方法

关键是在于 [root]/.deploy_git 文件夹。执行 hexo clean 的时候，只会把本地的 [root]/public 文件夹（存储待发布的 html 文件）给删除掉；到了执行 hexo d 的时候，会先把本地的 [root]/public 文件夹覆盖到本地的 [root]/.deploy_git 文件夹中作为中转，随后再把 .deploy_git 的内容 push 到远程仓库里。因此在本问题中，也能看到本地的 .deploy_git 文件夹里是用的大写的 Gallery 而不是小写的 gallery。

因此想要清除掉远程仓库中的缓存，具体方法就是把它删掉，然后再上传：

  1. 删除.deploy_git文件夹
  2. 执行hexo clean后，再执行hexo g，hexo g

上传错误 err 8

此外，有时候 hexo d 的时候会出现以下错误，HTTP/2 stream 0 was not closed cleanly: CANCEL (err 8)，那这个基本都是网络问题导致的上传被取消了，这时候换个手机热点就行了（学校的 wifi 有时候就是莫名网速慢）
