Hexo首行缩进

发表于2023-05-23|更新于2024-04-26|博客搭建
|字数总计:1.2k|阅读时长:4分钟

写在前面

首先，需求是让 Hexo 生成的静态网页的正文段中，每段的首行都能缩进。而 Markdown 本身秉持的主旨就是只管文本，不怎么管格式。在网页中，大概也有类似的文本、样式分离的模式，其中 CSS（层叠样式表）就是用来修改样式、控制网页外观的。因而，“首行缩进”的需求最终也应是由 CSS 实现。

在 Hexo NeXT 主题的作者问答板块中有提到说使用如下的 css 代码实现缩进，但一直苦于不知道要具体修改哪个文件。

直到找到了这一篇 CSDN 博文，才总算是解决了问题！而且意识到接下来可能还得去了解一下 HTML、CSS、JS 的关系才行。此外，之后的博客美化也可以很大程度地参考这篇博文。

回到本博客主题，要实现首行缩进，需要去主题文件夹中的自定义 CSS 文件里修改。新版本的 hexo 自定义 CSS 文件是 \themes\next\source\css\main.styl，自定义的部分加在 // Custom Layer 注释之后，加入上面提到的 CSS 代码即能实现。特别简单。

定义首行缩进 (text-indent)

在 NeXT 主题中，可以方便地在 main.styl 文件中定义注入的首行缩进代码，但是对于其他主题，就不一定是修改这个文件了，所以还是得稍微系统地讲解原理。

根据 CSS text-indent 属性定义文档，可以通过 <p> 标签的 text-indent 属性来定义首行缩进，如下所示。所以问题就变成了，如何给每个 post 的 html 文件自动增加这么一段定义了。

可以参考忘忧草的个人博客，具体方式是通过各 Hexo 主题定义的 inject 功能来手动注入 css 代码。

确认注入域

css 文件中会有不同的域，需要先找到博客正文的 <p> 标签所在的选择器/域，才能避免 text-indent 影响到其他的文本内容（如图标、div 标签）。每个主题文件可能会定义不同的 css 域，因此需要手动查看。

在本地通过 Hexo 命令构建网站源码 hexo g & hexo s。随后在 [Hexo Root]\public\css\index.css 中查看与博客正文相关的配置，例如在 Butterfly 中为 article-container p（我也不知道为什么下面这个 index.css 就能说明是 article-container 了，总之先这么记着吧！）

css 代码注入方式

根据上面的文件，可以判断对于 Butterfly 这个主题的 css 来说，需要将 text-indent 注入到 #article-container p 域中。要注意这个域是和 article-container ol p、article-container ul p 不一样的域。正确的注入方法是使用 element>element 选择器，例如我们改写为#article-container > p，这样只会选中父元素是 article-container 的所有 p 元素，从而有效地避免其他子元素中的 p 标签受到缩进而产生排版混乱。（不过今天试的时候，发现 Hexo 编译的时候不知道为什么，会自动把 > 转译成 &gt;，导致失效，所以最后还是把 > 省去了）
