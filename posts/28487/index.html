StableDiffusion的使用

发表于2023-05-22|更新于2024-04-19|搞点课外知识
|字数总计:3k|阅读时长:9分钟

要使用 Stable Diffusion WebUI，只需要运行 stable-diffusion-webui/webui-user.bat 文件就可以了（里面的配置都已经在之前的安装博文里面设置好了），将其中显示的本地 URL 复制到任意浏览器中，进入一下的核心产图界面。

大模型选择

stable-diffusion 生成图像时主要依据大模型，这个倒没什么好说的，普通人一般用的就是著名的 ChilloutMix，无脑选就行。

提示词设置

这里只讲文生图的部分。要知道，基于 AIGC 的主流模型 Diffusion 提出来的 Stable Diffusion，是一种将文本转换为图片（text-to-image）的模型，因而提示词（prompt）差不多是仅次于大模型的最重要的参数。当然，由于 AI 所特有的黑匣子特性，产图过程是封闭的，就和魔法一样，因而提示词也常被称为“咒语”。而这些提示词/咒语/ tags 不知道怎么使用的话，优先参考 civitai 里面的 demo 参数，其次就是像 NovelAI Tag、魔咒百科词典这种热心大佬总结的网站。

除去直接套用现成的提示词以外，还是有几个小知识点可以掌握的，这里主要参考一篇青松pine的 CSDN 博文以及 WebUI 的 Wiki 说明。

语法

  • 英文输入，支持自然语言描述（不过还是推荐使用多个关键词来描述）
  • 英文逗号分隔（前后的空格、换行不影响）不同的关键词，而用 | 分隔的关键词表示同比例混合（若有多个关键词，则是包含了所有关键词的任意排列组合），一个例子如下：
  • 提示词权重有两种写法，第一种是 (提示词:权重数值)，权重数值范围是 [0.1,100]，默认为 1，低于 1 为减弱，反之加强。第二种写法是外面嵌套括号，每套一层 () 括号是增强 1.1 倍，每套一层 [] 括号是减弱 1.1 倍，例如 (((提示词))) 表示加强 1.1^3=1.331 倍。
  • 渐变：[关键词1:关键词2:数字] 表示先按照关键词 1 生成图像，等迭代 x 步数（x 小于 1 时为总迭代步数的前 x% 步）后开始往关键词 2 生成图像。

书写要点

  • 不是越长的魔咒（提示词）生成的图片越厉害，请尽量将关键词控制在75个（100个）以内。
  • 越关键的词，越往前放。
  • 相似的同类，放在一起。
  • 只写必要的关键词。

反向提示词(Negative Prompt)

AI 出的图是可能很丑的（比如三头六臂），需要用文字描述不希望在图像中出现的东西。反向提示词在 AI 中的大致流程是这样的：

  1. 对图片进行去噪处理，使其看起来更像你的提示词。
  2. 对图片进行去噪处理，使其看起来更像你的反向提示词（无条件条件）。
  3. 观察这两者之间的差异，并利用它来产生一组对噪声图片的改变
  4. 尝试将最终结果移向前者而远离后者
  5. 一个相对比较通用的负面提示词设置

Lora 模型

这个一般是在正向提示词（Prompt）里输入的。使用时与 Textual Inversion 一样，当做一个单独的关键词输入，不过使用时需要用尖括号，而且不能放到 Negative Prompt 里（Textual Inversion 则可以放在 Negative Prompt 和 Prompt 里），具体语法为：<lora:filename:multiplier>，其中 filename 是下载的本地 LoRA 文件的实际文件名，multiplier 一般是 [0,1] 间表示该 LoRA 权重的数值。使用 LoRA 时不支持用 | 连接进行混用，多个 LoRA 只能使用英文逗号连接。

其他绘图参数

采样方法 Sampling method

关于采样器方法的选择，可以参考别的大佬做出的详细总结：stable diffusion webui如何工作以及采样方法的对比。常用的采样器如下：

  • Euler a ：富有创造力，不同步数可以生产出不同的图片。 超过30~40步基本就没什么增益了。
  • Euler：最最常见基础的算法，最简单的，也是最快的。
  • DDIM：收敛快，一般20步就差不多了。
  • LMS：eular的延伸算法，相对更稳定一点，30步就比较稳定了
  • PLMS：再改进一点LMS
  • DPM2：DDIM的一种改进版，它的速度大约是 DDIM 的两倍

采样迭代步数 Sampling Steps

一般使用默认的 20 步就行。AI 绘画的大致原理是先生成噪声图片，然后再逐步往提示词方向调整图片，迭代步数越多，每次调整量就越小越精确。大部分采样器超过 50 步之后的意义都不大。下图是一组提示词在第 1 步到第 20 步的图像变化过程，可以看到差不多是 10 步之后结果就收敛了。

面部修复 Restore faces

对生成图片任务的面部进行修复，对真人效果好，很占显存。

可平铺 Tiling

生成可以往左右、上下连续拼接的图像

高清修复 Hires. fix

作用是将图片按照设置的放大倍率重新绘制图片内容。在勾选了 Hires.fix 选项后，会多出两行的选项

  • 放大算法：不知道选什么就选“ESRGAN_4x ”
  • 高清修复采样次数 Hires steps: 20 就行
  • 重绘幅度 Denoising：放大后细节的修改程度，从 0 到 1，数值越大，AI 随机程度越大，也就越偏离原图， 一般 0.5 - 0.7 就行
  • 放大倍率：即图像放大倍率，数值越大，对显卡能力要求越高

宽&高 Width & Height

单位像素。适当增加尺寸，AI 就会试图填充更多细节。较小的尺寸（小于 256 X 256）会让 AI 没地方发挥而导致图像质量下降；较大的尺寸（大于 1024 X 1024）也会让 AI 因乱发挥而导致图像质量下降。4GB 显存的极限大概是 1280 X 1280。常见的模型基本都是在 512 X 512、768 X 768 的基础上训练的，因而自己用的时候也可以用这种尺寸的配置，再用 Hires.fix 来生成高分辨率图像。

生成批次 Batch count / n_iter

用同样的参数循环跑的次数，每次产图时的随机种子都不一样，如果指定了随机种子，则相邻批次产图的种子数＋1。该选项是为了产生多个备选图像以供选择（毕竟 AI 不能保证每次出图都是好图）

每批数量 Batch size

每个批次下同时生成的图片数量，每次产图时的随机种子都不一样，每张的种子数＋1。与生成批次不同的是，增加这个值可以并行计算。例如，512 X 512 的图，使用 SD1.4 模型，Euler a 采样器，4G 显存可以并行 2 张图，8G 显存可以并行 8 张。

提示词相关性 CFG Scale

图像域提示词的匹配程度。一般取 [5,15] 的值，常用的值是 7，9，12，过高的值会让图像色彩过于饱和而降低了图像质量（此时，也可以通过增加采样迭代步数来抵消画质劣化）。

随机种子 Seed

AI 绘图的原理是先随机出一个噪声图片，再进行后续计算，该噪声图片作为初始输入就会影响最终的产图，而随机种子就是控制该随机过程的参数（计算机不存在真随机）。如果保持随机种子不变，则相同的模型、参数会大概率生成相同的图像，但要注意的是，即使采用完全一致的参数、模型，不同型号的显卡也可能会生成完全不同的图。

上图中，骰子图标表示快速输入 -1（随机值），绿色图标表示自动填写最近生成的图片的种子值。勾选右侧选项后会出现额外参数设置，其中差异随机种子 Variation seed 可以是 -1，也可以是固定值；差异强度 Variation strength 表示最终随机数是更偏向随机种子还是差异随机种子，值越大则最终的随机数越接近差异随机种子，值越小则越偏向随机种子。
