MATLAB-AppDesigner程序打包

发表于2023-10-15|更新于2024-04-29|学术模块
|字数总计:1.4k|阅读时长:4分钟

程序打包

写好 .mlapp 文件后若想直接发送给别人使用，至少就得要对方的运行环境与自己的 MATLAB 一致，且也需要打开 MATLAB 才能使用。倘若想要加密为一个 .exe 那种的可执行文件，则需要用到 App Designer 的程序打包功能。话先说在前面，MATLAB 是主要为计算所设计的平台，不是专业做桌面应用的，其打包得到的可执行文件其实不太好用，主要有两点：

  • 计算效率慢。.mlapp 的运行速度就会比 .m 文件慢一点，封装成 .exe 之后会更加慢。
  • 程序运行的依赖东西多。主要是依赖 MATLAB 的 Runtime 库，要么需要用户也安装了，要么一块打包进 .exe 中（要额外占用 1GB 的内存空间）

在 App Designer 主界面的左上角菜单栏中的共享里，一共有三种选项，对应三种程序打包方式和可执行文件的交付场景，下面也大概介绍一下这三种方式。

  • MATLAB App：需要使用方也安装并打开 MATLAB 才能使用
  • 独立桌面 App：需要使用方安装 MATLAB Runtime（约 1GB）
  • Web App：网络访问部署在服务器端的 App

法一：MATLAB App

首先点击上述的共享——MATLAB App，随后在弹出的框里填写所需的信息，然后点打包。

随后就会在目标文件夹里生成一个 【AppName.mlappinstall】 的文件，双击并安装后，就能在 MATLAB 主页里的 App 处找到对应的入口了。要使用的时候，打开 MATLAB 并且双击图标就可以，还是挺方便的，要卸载时再右键卸载即可。

法二：独立桌面 App

两种打包方式

这里要注意的一点是，该打包方法是用于在没有安装完整 MATLAB 环境的电脑上运行的，实际上程序运行是需要一个有着 MATLAB 基础功能的软件，即 MATLAB Runtime，而围绕这个 Runtime 是否包含在打包的程序里，就会有两种打包方式，下面也分成两部分讲：

  • Runtime downloaded from web：用户自行从网络中下载 MATLAB runtime
  • Runtime included in package：程序打包时就将 MATLAB runtime 包含在程序里面

关于runtime安装

MATLAB Runtime（大概 3GB）就是 MATLAB（大概几十 GB）的基础功能版。在 MATLAB 命令行输入 mcr 可以查看是否有安装 Runtime。如果没有，则可以通过 compiler.runtime.download 下载，不过这个一般都很慢，即使挂了梯子可能也会连接超时，所以一般采用网页下载的方式。

可以在输入了 compiler.runtime.download 之后按 Ctrl + c 中止命令，此时可以看到对应的下载网页，在浏览器中输入并下载。

下载好后压缩包后可以选择解压然后直接安装，也可以选择不解压，通过配置 MATLAB 编译器路径来实现编译的时候调用。

关于MATLAB编译器检查

在命令行窗口输入 mex -setup 以检查 MATLAB 是否配置好了对应的编译器，如果没有则上网找教程安装吧！（一般都会有的）

runtime from web

首先点击上述的共享——独立桌面 App，随后在弹出的框里填写所需的信息，然后点打包就行。

选择该方式打包， 会生成三个文件夹：

  • for_redistribution：包含了 MyAppInstaller_mcr.exe 文件，该安装包里包括了程序的安装以及通过 Web 安装 MATLAB Runtime 的执行程序
  • for_redistribution_only：打包程序的 .exe 文件（相当于是上面安装好之后得到的可执行文件），可以直接运行，但前提是运行环境已经安装过 MATLAB Runtime
  • for_tesing：开发者的测试文件

在使用时，可以只发送 for_redistribution 文件夹里的内容，用户端双击点开后会进入安装界面。

若用户端未安装过 MATLAB Runtime，则会自动安装（速度会比较慢）。安装好之后就能得到独立运行的桌面 App 了。

而如果用户已经安装过 MATLAB Runtime，则可以只发送 for_redistribution_only 文件夹并直接运行里面的 .exe 文件。

runtime included

选择 Runtime included in package 的程序打包方式，在打包过程中就会多了一个图标，表示把 Runtime 给压缩了进去。

打包好之后，也会生成三个文件夹，此时 for_redistribution 文件夹就比较大，里面包含的 MyAppInstaller_mcr.exe 安装程序就包括了打包的程序以及 MATLAB Runtime 的安装程序，此时可以离线安装。

同理，在使用时，可以只发送 for_redistribution 文件夹里的内容，用户端双击点开后会进入安装界面，等待离线安装完成就行。而如果用户已经安装过 MATLAB Runtime，则可以只发送 for_redistribution_only 文件夹并直接运行里面的 .exe 文件。

法三：Web App

程序打包好后会生成 .ctf 文件，该程序文件需要配合 MATLAB Web App Server 使用，相当于在服务器端部署了一个程序应用，等待客户端远程调用，可以参考 Matlab 官方文档。这里就不讲了，我也暂时没碰到这种应用场景，或许之后自己搭一个云服务器的时候会用到吧。
