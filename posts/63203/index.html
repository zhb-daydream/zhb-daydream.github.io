【vue前端】开始一个新项目

发表于2025-07-18|更新于2025-08-04|vue前端
|字数总计:651|阅读时长:2分钟

最近想做一个圈子里的学术家谱，这个分为结构数据存储和网页前端渲染两部分。对于动态网页的部分，由于我基本没有编程经验，所以不太想从传统的前端三剑客（HTML+CSS+JavaScript）从头学起，感觉可以学点新一点的东西，就是vue.js（Vue 1/2/3分别是2014年、2016年、2020年发布的，目前默认都是用Vue 3）做框架，并且用vite（https://cn.vitejs.dev/guide/）做网页构建。

配置环境

首先需要按照node.js。这是JavaScript运行时的环境，在安装node.js的时候会自动安装npm（Node Package Manager）这一个包管理器工具，后续可以通过npm命令区安装其他第三方库了。npm的注册表包括了100万个开源包，这意味着不需要自己重复造轮子。

安装过程这里不列了，一般是通过NVM（Node Version Manager）或者官方下载安装包两种方式，但检查方式可以通过命令行的 node -v 与 npm -v 来检查版本。总之，差不多能用就行了，别太纠结这些细枝末节。（❗注：vite需要20.19+、22.12+以上的Node.js版本，有的第三方库可能还需要更高的版本，所以请尽量用新的Node版本）

创建vite+vue样例

使用万能的vscode作为编辑器，新建一个 viteProjects 文件夹，并在终端里通过 npm create vite@latest 在该文件夹下创建一个 vueDemo 的项目，期间先后选择 Package name——Framework（选择vue框架）——Variant（选择TypeScript项目语言，它可以兼容JavaScript，同时可维护性更高）。

随后就完成了初始样例的搭建，此时可以发现有一个新的 vueDemo 文件夹以及里面的一堆文件。

接着通过 code .\vueDemo 命令来让 vscode 进入到该项目里面，并更新左侧工作区的文件夹。并先后运行 npm install（安装package.json文件中所定义的依赖库）和 npm run dev（编译项目）命令，从而得到 vite+vue 编译后的样例网页。此时 vite 会开启一个本地服务器，通过对应的网址就能访问到编译的动态网页了。

后续通过 npm run build 命令就会创建一个 dist 的文件夹，把里面的所有文件部署到相应的平台（例如GitHub主页），就能完成静态网页（可以包含CSS/JavaScript等的动画交互，但其内容不会随着时间/服务器而改变）部署了。
