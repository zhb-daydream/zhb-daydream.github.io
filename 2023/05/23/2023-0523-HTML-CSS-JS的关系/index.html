HTML-CSS-JS的关系

发表于 2023-05-23 更新于 2023-06-09 分类于 博客搭建
本文字数： 1.5k 阅读时长 ≈ 5 分钟

一般下，Web 前端有三剑客：HTML、CSS、JavaScript，这是三种不同的技术，实际应用时还需要相互搭配使用。

HTML

即超文本标记语言（Hyper Text Markup Language），是一种不严谨、简单的描述性语言。它用各种标签将页面中的元素组织起来，告诉浏览器如何显示内容。其中，“不严谨”是指，HTML 标签使用标准多，即使不加结尾也可以正常运行。

CSS

即层叠样式表（Cascading Style Sheets），是用来修饰 HTML 文件样式、控制网页外观的语言，负责字体、颜色、位置等外观的描述。其中，CSS 的“层叠”（cascading）表示“样式”（style）作用在同一个 HTML 元素的同一个属性时，具有优先级的层次结构，会依据层叠规则（权重）来处理冲突，优先选择应用权重更高的 CSS 选择器所指定的属性。

在 Hexo 博客中，要修改的 CSS 文件就是各主题文件里面的对应文件。具体到 NeXT 主题，就是 \themes\next\source\css\main.styl 文件。

JavaScript

一种嵌入到 HTML 页面中的脚本语言，简写为 JS，是用来做交互的。由浏览器一边解释一边执行，既可以运行在客户端（前端）也可以运行在服务器端（后端），用于扩展文档交互能力，使静态的 HTML 具有一定的动态功能（比如表单提交、动画特效、弹窗等）。注意，JavaScript 与 Java 完全没有关系，两者只是名称刚好相近（就像雷锋和雷锋塔）。

Hexo 中的 JS 文件主要就是 layout 文件夹中的各文件，具体到 NeXT 主题，就是 \themes\next\layout 里的各个 .swig 文件（swig 是一种 JS 模板引擎，见下文）。

与 Node.js 关系

JavaScript 是一种主要用于前端 Web 开发的脚本语言，而 Node.js 是一个后端用于运行 JavaScript 的环境，是适合初学者的流行且轻量级的 Web 框架，使得用户可以在任何浏览器上使用 JavaScript（包括前端、中间件、后端，以及 Linux、macOS、Windows）。

更具体一点，目前支持 JavaScript 的引擎（JS 引擎独立于它所在的浏览器，只不过各浏览器厂商倾向于用他们自己提出的 JS 引擎）主要有 Safari 的 JavaScript Core、FireFox 的 Spider Money、Google Chrome 的 V8 三种。而提供运行 JavaScript 环境的 Node.js，本质上用的就是 V8 引擎。

Swig 模板引擎

模板引擎本身是一个跨领域、跨平台的概念，在 Asp、PHP、C#、JavaScript 开发时都会用到模板引擎技术，其作用是将业务数据代码（内容）与界面逻辑代码（显示）分开，从而提高开发效率，降低项目维护难度。

具体到 Web 开发中的 JavaScript（JS）的模板引擎而言，JS 模板引擎是用于动态生成 HTML 的工具，负责将数据和模板结合。常见的 JS 模板引擎有很多，基本都可以直接通过 npm install 安装，以下就随便列几种：

  • EJS（Embedded JS）
  • Jade
  • Haml
  • artTemplate（腾讯的）
  • Swig🌟

其中，Hexo 默认使用 Swig 模板引擎，上手会简单一点。此外也可以另外安装插件了获得 EJS、Haml、Jade 的支持，Hexo 会根据模板文件的扩展名来决定所使用的目标引擎，例如：layout.ejs，layout.swig。
