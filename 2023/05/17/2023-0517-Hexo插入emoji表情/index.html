引入方式

在 Hexo 中引入 emoji 表情，很多博文首先推荐的是把默认的渲染器 hexo-renderer-marked 卸载换成 hexo-renderer-markdown-it 并在 _config.yml 里做相应声明，前者我在捣鼓公式渲染的时候就发现会与 hexo-filter-mathjax 插件冲突了，换成后者之后，果然也出现类似的问题（区别在于，此时只有复杂一点的行间公式渲染不出来）。

由于一个是 renderer，另一个是 filter，可能两者重量级不一样导致冲突（奇特的脑回路）？不管怎样，经过一番搜寻，找到个 hexo-filter-github-emojis 的插件，即识别文本中对应的 markdown emoji 语法（::,例如：:smile:）转变成对应的 emoji 表情，可能由于同为 filter，只涉及一些基础的文本替换，所以不会与 hexo-filter-mathjax 插件冲突。

背景补充

emoji 表情是目前网络用的比较多的表情符号（划重点：本质是符号，不是图片），可以在任何支持文本输入的地方使用，目前由统一码联盟（Unicode Consortium）进行规范和管理，即每一个 emoji 在 Unicode 标准里都有其对应的码位，因而也保证了它可以在不同设备和平台上通用（当然，同一个 emoji 在不同设备/平台上可能会显示不一样）。Emoji14.0（2021-0914 推出）里就已包含 3000 多个符号，后续还会不断更新其他版本的 emoji。

想在 Markdown 中引入 emoji，只需要用两个冒号，并在中间加入对应的 emoji 编码（英文字符）即可，例如 :smile: 对应为 😄。
