Hexo部署到Github主页

发表于 2023-07-18 分类于 博客搭建
本文字数： 1k 阅读时长 ≈ 3 分钟

隔了这么久，发现还没记录怎么把博客部署到 Github 上，考虑到之后可能准备换源换到国内的 Gitee 或者学校个人网盘里，想想还是补一下吧。

新建Github仓库

首先登录到 Github 后，创建一个 username.github.io 的仓库，注意这里的 username 一定要与用户名相同，之后才能通过 http://username.github.io 来访问，如果创建了 username 以外的仓库名并上传了博客，例如 wrongname.github.io，那么博客实际上是部署在 http://username.github.io/wrongname.github.io。所以，如果想直接通过 username.github.io 访问，就需要该仓库名与用户名一致。

创建仓库时，仓库属性应设为 public，这样才能使用 Github Pages 服务，即公开访问个人主页。

设置公钥

先在本地通过 git 生成密钥（省略），然后去 Github 网页中刚创建的 username.github.io 仓库中找到设置（settings）——部署密钥（Deploy keys）——添加密钥（add deploy keys）。该操作主要是给本地的 git 部署设置该仓管库的使用权限的，当然，也可以直接去 github 的个人设置界面中上传公钥。

设置Hexo配置文件

在博客根目录的 _config.yml 文件中设置部署的类型为 git、仓库为 https://github.com/username/username.github.io.git。

此外，还需要在该文件中设置 url 如下所示追涨杀跌，这个操作是用来设置网页文件夹的相对路径的。

通过Hexo命令部署

先通过命令行的 npm 安装 deploy-git（即 hexo deploy 的命令），这样才能用命令行自动将静态文件部署到 Github。

然后就是 Hexo 三连，即可自动部署。部署成功后过一会就会更新了，如果部署不成功，有时候是因为网络问题，多重复几次就行了。

其他

除了 Github 部署以外，其实还有域名设置、FTP 上传网盘等功能，这些就等我以后用到再补充吧。

sci投稿时的transfer desk

发表于 2023-07-17 更新于 2023-07-18 分类于 学术模块

Transfer Desk

sci论文在被拒的时候，有些期刊的编辑把被拒的稿件移交到 Transfer Desk，并由 Transfer Desk Team 帮忙把被拒的稿件转投到该杂志社旗下的其他期刊中。

Springer Nature 杂志社就提供这种 Transfer Desk 的转投服务，会有专门的期刊投稿顾问（Editorial Submission Advisors，ESAs）团队来审核需要转投的稿件，通过期刊匹配以及结合其经验、投稿人的投稿偏好、编辑或审稿人的意见来提供推荐转投的 Springer Nature 旗下的期刊。

转投流程

（1）Springer Nature 的 Transfer Desk 转投服务主要是通过电子邮件往来完成推荐、转投流程的。一般被拒了之后，编辑在拒信的底部会告诉你稿件已被转到 Transfer Desk 且 48 小时内会有 Transfer Desk 相关的邮件发给你。

（2）接着就会有 Transfer Desk Team 发来的邮件，说他们会处理，并给出相应的一些期刊推荐以及推荐原因，推荐的时候会列出其影响因子和引用因子，不过一般都比被拒稿期刊的差一点。

同时在邮件下方会列出，如果决定好想转投哪个期刊，就直接回复邮件并附上期刊名字即可。

（3）回复邮件意愿和期刊名称之后，多数情况可自动完成。一般不要求对转投稿件进行修改。

（4）转投后作者会收到一个链接，可用于上传一份更新后的稿件、cover letter等内容。一般也建议作者针对转投的期刊修改cover letter。大多数情况下，转投服务可以自动将稿件从被拒期刊转移至转投的新期刊，并将自动填写大部分作者信息等元数据。但在这一过程中，作者依然有机会在完成新一次投稿前对自己的稿件进行修改。个别期刊可能会有一些特殊的排版要求（如声明、副标题等），请在重新投稿时浏览新投稿期刊的具体投稿要求。多数需要进行同行评审的期刊有标准的参考文献格式，请根据具体情况酌情调整。

转投利弊

（1）审稿人建议转投的期刊大概率水平低于文章质量，有些期刊还是高版面费，基于上述情况转投通常会加快发表进程，录用率也会有所提高。

（2）相对于拒稿重投，转投会更简单些。如果被拒稿换刊重投，作者需要根据新期刊的投稿须知修改格式，而且需要登录投稿系统重新提交一遍投稿文件。而转投的期刊不需要返工，有些只需要回复同意转投，期刊就会自动把文章转过去，或者同意转投后自行提交系统，同一出版商期刊的格式基本相差不大，几乎不需要改动。

（3）转投弊端就是文章有可能会发表在水平一般的期刊。另外，论文不一定会被转投的期刊接收。如果没有被接收，还需要换刊重投更加浪费时间。

sci论文的利益冲突是什么

发表于 2023-07-17 分类于 学术模块
本文字数： 757 阅读时长 ≈ 3 分钟

在sci论文投稿的时候，一般都会要求在正文或者cover letter中描述利益冲突（conflict of interest），而大部分人都是直接写 "We have no conflicts of interest to disclose"。那么，这个利益冲突具体是什么东西呢？

利益冲突的定义

利益冲突（conflict of interest）是指个人或组织的利益可能会影响论文客观性的情况。因为在论文写作中，难免会牵扯到资金支持、学术合作等，如果在撰写论文时资金资助来源较为复杂，而其中某项资金来源同其他相关研究有一定的关系，那么其研究数据乃至整个论文的客观性就会受到怀疑。这一类的潜在利益如果不能在科研发表工作时列出来，则读者可能会觉得自己被误导或欺骗，降低了科研过程的透明度以及期刊的可信度。因而为了防患于未然、保证学术研究的公正性和客观性，编辑会要求作者在投稿时准备好利益冲突声明。

利益冲突的类型

常见的利益冲突类型有两类：

  • 财务关系：最常见的利益冲突。如雇主、股东、赞助商支持、旅费津贴、专利等，是最有可能破坏期刊、作者甚至科学本身的信任的因素。为此，一般要说明资助或津贴的确切金额、对研究起到的作用和影响
  • 非财务关系：包括个人的家人/合作伙伴、学术竞争关系、意识形态与信仰等。需要阐明这些关系具体是什么，以及可能会引发怎样的潜在利益冲突。
