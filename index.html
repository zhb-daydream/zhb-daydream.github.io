ChatPDF使用方法

ChatGPT 是 NLP 的一个大模型，专门用于文本处理与分析，而在社会生产中，一个很直接的地方就是科研论文的统计与分析，ChatPDF 就是这么个在线免费的论文辅助阅读工具。

上传论文PDF

首先，打开网址：https://www.chatpdf.com/，上传下载好的 PDF 论文文件（也可以用 URL，不过好像非开源的论文 URL 大部分都识别不出来。

提问

读取成功后，会列出论文的一些基本信息。

然后就可以提问了。

基本用法就是这么简单，好像也没什么特殊的了，主要就是用来精一点的略读（毕竟有些原理性的回答还是不太行），了解个大概的论文背景，尤其是 introduction 部分，全是文本、没有公式，特别适合用 AI 来总结。

中文版ChatPDF

写博客的时候发现上面这个英文版的 ChatPDF 响应速度大大变慢了，而且也没法正常上传读取 PDF 并浏览以前的记录了。翻了一下，有个深圳公司开发的中文版的 ChatPDF （https://www.chatpdfs.cn/）好像还行，但就是要微信扫码登录，不知道以后会不会收费。

妙鸭相机等AI写真软件原理

发表于 2023-08-13 分类于 搞点课外知识
本文字数： 1.7k 阅读时长 ≈ 6 分钟

2023 年 7 月 17 日正式上线的妙鸭相机在最近挺火的，它主打的就是一个方便快捷，只要 9.9 元以及上传大概 20 张图片，就能生成个人的数字分身，并一键生成商务照、证件照等各种风格的个人写真照，在微信小程序上搜一下就能用了。

像妙鸭相机、45AI 写真这一类的 AI 小程序算是 AI 绘画技术风口下的第一批商业产品，这一类互联网产品只要底层的软件写好，使用量上去之后，边际成本是几乎为零的，剩下的事情主要就是营销推广以及用户留存率了。不过这里主要还是想了解一下这种 AI 写真软件的原理，理论上就和 Stable Diffusion 中训练 LoRA 模型是相通的。

原理

“投入少量图片，经过一定训练时间生成自定义的不同风格的写真照片”，该效果描述基本就是今年年初火起来的 LoRA（Low-Rank Adaptation of Large Language Models（大语言模型的低阶适应） AI 画图模型，其效果就是在 Checkpoint 等大的主模型基础上，利用少量数据训练处一种画风/IP/人物，实现定制化需求。正因为它所需的训练资源比其他模型要少很多，所以非常适合社区使用者和个人开发者。

而妙鸭相机等 AI 写真软件就是基于开源的 Stable Diffusion 来进行 LoRA 模型的精调，相当于是把科学上网、找资源、本地部署 Stable Diffusion、配置插件环境、输入调试提示词 Prompt 等等这些步骤全都集成化到一个在线使用的小程序上，而且操作简单、收费低（9.9 元），确实能直击用户痛点。

妙鸭相机使用方式

使用的时候也很简单，首先去微信小程序（不需要下载 APP）搜索并点开。

上传单张照片

然后需要使用一张图片作为输入，按照 Stable Diffusion 的逻辑，这一张照片应该就类似于一键换头、局部重绘之类的输入图片。

这里提到“婴儿、老年人、眼镜等特征效果有不稳定性，我们正在努力优化”，如果该软件用的是开源的 Stable Diffusion 以及相应的 Check Point 大模型，那么该团队实际上是没有能力去优化这些特征效果的，可能就是圈一波钱就跑路那种模式。

补充20张正面照

这里使用的多张照片，猜测实际上就是用来重新训练 LoRA 模型。像是美图秀秀、B612 等软件会要求更少的 3-15 张，生成的图片可能没那么与本人相似。此外，这些训练数据也有一定的质量要求，要保证面部特征清晰、单人、多角度等，才属于优质数据，否则训练出来的 AI 模型很可能也不行。

付费并等待

挑选并上传完图片后，就会提示付费并生成图片。（真会玩，要等用户投入了时间之后再提示要花钱，在沉没成本下或许大部分人都会选择掏个 9.9 元吧）。

付费成功后，一般还要排队等待。根据科技每日推送的一篇博文，要大概 3 小时。

自定义写真

等待结束后，能看到制作好的数字分身以及几张范例照片。

后续就能够根据一些模板调整出对应的写真照片了。

软件评价

隐私泄露

要训练 AI 模型就需要上传个人照片，而这个过程是在云端数据库进行的，并不能在客户端的本地运行（其实也合理，毕竟主打一个方便快捷，那麻烦的运算操作肯定都统一放在云端）。而这必然就会引起个人隐私数据泄露的风险，而且最初的《用户须知》里面的授权许可款项，让用户授予了软件开发方的很多权利。

尽管后续发了一份道歉声明以及删除承诺，但其原理决定了这种隐患无法被消除。

生成效果的稳定性

AI 本质上还是个黑盒子，虽然目前确实能生成出让人眼前一亮的照片，但也经常会出现照片细节不好的问题。但对于用户来说，并不会理会那么多原理性的细节，只要出来的图不好、生成错误、客服跟不上，那就很容易给差评。

封神演义的等级制度

发表于 2023-08-13 分类于 搞点课外知识
本文字数： 1.5k 阅读时长 ≈ 5 分钟

最近看了《封神榜-第一部》的电影，感觉商周大战+（背后）阐截大战这种世界观很有意思，再加上现在网文里有一大流派叫“洪荒流”，大概就是以《封神演义》为世界观的一类小说，而我之前看过的很喜欢的一篇叫《我的师兄实在太稳健了》就是这个流派里面的，里面看阐截教如何勾心斗角、人教如何试图把西方教给拉进劫中、纣王帝辛如何逆天道（灭商兴周）而行，很有意思。所以还是想略微把这个世界观给列一列，顺便熟悉一下之前学的 Mermaid 写法。

何为封神

《封神演义》（俗称《封神榜》，又名《商周列国全传》、《武王伐纣外史》等）是明代的一个虚构长篇小说，共一百回，前三十回讲商超末代君主纣王的暴虐，后七十回讲商、周两国的战争，以纣王自焚、武王夺取天下并分封列国为结局，中间掺杂一些女娲、妲己、元始天尊、姜子牙、申公豹等神话人物，这些神话人物是属于中国佛道神话体系里面的，与西游记里的也有一定关系。

故事起源于封神榜，因为（1）阐教十二金仙犯了杀伐之厄、（2）昊天上帝成立的天庭缺人、（3）人间正值商周王朝更迭，共三个背景原因，于是鸿钧在紫霄宫中召开大会，与三教掌门讨论并一致决定由阐教二代弟子姜子牙执掌封神榜，从三教中选取 365 人封为天庭八部三百六十五位正神。在此前，三教各弟子属于仙，是自由自在的，而若上了封神榜、进了天庭，则为神，是属于有编制的员工。虽然两者都属于神仙，但还是有很大的差别，因而大家都不想上封神榜。最后也演变成，只有身死道消的仙人才会上榜，整个《封神演义》的故事便是围绕这么个封神来展开的。

本文字数： 3.5k 阅读时长 ≈ 12 分钟

在人类文明的早期，人类还处于探索世界的阶段，古老先辈们会试图描绘、想象世界的真实样貌，当这种想象逐渐升级为一种对于超自然神祇（qi，第二声）进行崇拜的集体行为，同时一套可以自圆其说的世界观建立时，便是一个神话体系了。要注意的是，神话还只是一种民间文学，属于一个逻辑自洽的世界观，而宗教则是一种特殊的社会意识形态，两者还是有点微妙的不同。

想起《文明6》里面的旁白：“从水下第一个生命的萌芽开始，到石器时代的巨型野兽，再到人类第一次直立行走，你已经历许多。现在，开启你最伟大的探索吧：从早期文明的摇篮到浩瀚星宇”，人类文明的演化总是多样化的不同地区就会有不同的神话体系，这里也主要是简单列一列我听过的几个神话体系（然后就可以去和别人吹牛逼了）。

基本介绍

  • 先秦时期华夏先民所创造的早期神话传说，在经过后人传承与改写后形成大致的脉系
  • 主要文献包括《山海经》、《尚书》、《淮南子》、《史记》、《楚辞》等
  • 多神崇拜、英雄史诗

everything搜索软件的使用和原理

发表于 2023-08-06 分类于 搞点课外知识
本文字数： 2.3k 阅读时长 ≈ 8 分钟

现在的面向大众的电脑硬盘容量越来越大，特别是最近长江存储颗粒的固态硬盘，2T的容量只要惊人的350块。但硬盘容量上去了，文件数量变多了，文件搜索也是一个大问题。除了使用规范化的文件夹命名、打开文件夹手动查找文件的方式以外，有时候还会使用Windows自带的文件搜索器，但这个搜索往往很耗时。所以这里就介绍一个可以充当Windows搜索功能的上位替代品的软件。

Everything

Everything是Voidtools制作组开发的文件搜索工具，官网描述为基于名称的实时文件和目录定位（Locate files and folders by name instantl），在2004年时创立项目，2008年推出了完善的1.0版本，目前最新的是2023年5月26更新的1.4.1.1024版本。

Everything（也称Everything Search Engine）软件堪称搜索界的神器，它是一个第三方的搜索软件，搜索速度特别快，而且CPU占用特别低，例如使用Windows系统自带搜索功能去搜索全盘，CPU、磁盘读写可能都会占用100%，且需要持续若干分钟，此时Everything只占用0.1%的CPU和磁盘读写，且只需持续个0.001秒，即使待搜索文件有几百万个，且对结果排序（例如按大小排序）都是毫秒级响应，使用时也不需要管理员权限。

使用方法

使用的时候也很简单，从官网的下载地址获取免安装的便携版，解压之后直接运行.exe文件，在首次或者隔一段时间后就会花个大概10秒钟进行硬盘数据的扫描（我的固态电脑大概366万的对象用了10秒不到的时间完成索引库的建立）。

如果是第一次运行，可能会弹出以下提示框，选择安装“Everything”服务即可。

接着在搜索框内输入待检索的内容（可以是文件/文件夹名字，也可以是.pdf等后缀扩展名），就能几乎是秒出排序结果了。

除了上面这种最近的搜索功能以外，还支持正则表达式、自定义的高级搜索等功能，不过这些我都没怎么用过。

快速搜索原理

Everything在第一次打开使用的时候，它会扫描整个硬盘，然后建立一个索引数据库（大小和电脑文件的数量有直接关系）。索引数据库建立之后，每次搜索文件，everything就在它的数据库里搜索，而不需要重新扫描整个硬盘，这也是everything和Windows系统自带搜索的本质区别。而everything之所有能快速地建立起这么小的索引数据库，主要得益于Windows对NTFS（New Technology File System，一种磁盘格式，除此之外还有FAT32等磁盘格式/文件系统）的支持。

MFT数据表

对于NTFS文件系统下的所有数据，都会有一个MFT数据表（Master File Table），该表记录了NTFS磁盘上所有文件的文件信息，且访问速度很快，Windows提供了各种API给开发者去直接调用并访问MFT数据表，不需要遍历文件系统就能获取当前卷/磁盘里的所有文件名称和路径。

USN Journal

除了MFT数据表以外，NTFS文件系统还有一个日志功能，即所有对文件系统有修改的操作都会被记录在USN Journal日志文件中，且该日志文件是实时、同步更新的。

Everything实现

Everything搜索引擎主要就是利用了NTFS文件系统中MFT和USN Journal的特性：

  1. 在程序启动时，扫描所有NTFS卷/磁盘的MTF表
  2. 将MTF表里面的文件名称以一种利于字符串检索的算法形式化存储在索引数据库中（在退出程序后就会看到有个大概一百多MB的Everything.db文件）
  3. 后台运行时，Everything会监控NTFS卷的USN Journal日志文件，实时更新该索引数据库
  4. 用户搜索文件时，Everything利用其字符串查找算法，在索引数据库里查找

从上面的实现过程可以看出，everything的搜索速度快、占用小，除了得益于其索引数据库生成算法和字符串查找算法以外，主要还是利用了MTF表的易于访问的特性。也正是因为这样，everything只能检索NTFS磁盘的文件，而不能检索其他如FAT32、exFAT等文件系统。

虽然everything比Windows自动的搜索功能快几个量级，但Windows也没有采取这种方法，可能主要原因还是因为文件系统的兼容性不足（仅局限于NTFS系统），且不支持搜索文件内容、附加属性。此外，微软自家的ActiveDirectory软件其实也有在使用MTF特性来快速检索文件。
