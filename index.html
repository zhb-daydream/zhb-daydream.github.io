五个世界神话体系

发表于 2023-08-06 分类于 搞点课外知识
本文字数： 3.5k 阅读时长 ≈ 12 分钟

在人类文明的早期，人类还处于探索世界的阶段，古老先辈们会试图描绘、想象世界的真实样貌，当这种想象逐渐升级为一种对于超自然神祇（qi，第二声）进行崇拜的集体行为，同时一套可以自圆其说的世界观建立时，便是一个神话体系了。要注意的是，神话还只是一种民间文学，属于一个逻辑自洽的世界观，而宗教则是一种特殊的社会意识形态，两者还是有点微妙的不同。

想起《文明6》里面的旁白：“从水下第一个生命的萌芽开始，到石器时代的巨型野兽，再到人类第一次直立行走，你已经历许多。现在，开启你最伟大的探索吧：从早期文明的摇篮到浩瀚星宇”，人类文明的演化总是多样化的不同地区就会有不同的神话体系，这里也主要是简单列一列我听过的几个神话体系（然后就可以去和别人吹牛逼了）。

中国先秦神话

基本介绍

  • 先秦时期华夏先民所创造的早期神话传说，在经过后人传承与改写后形成大致的脉系
  • 主要文献包括《山海经》、《尚书》、《淮南子》、《史记》、《楚辞》等
  • 多神崇拜、英雄史诗

主要神系

  • “炎黄史诗与虞夏英雄史诗”神系：伏羲(太昊)、女娲、燧人、祝融、共工、颛顼、帝俊、神农、黄帝(轩辕)、刑天、尧、舜、禹、后羿、夸父等。
  • “西王母传说”神系：西王母。
  • “蓬莱仙话传说”神系（春秋战国后期流传）：蓬莱仙人群体。
  • “南方楚地九歌”神系：主要神话人物:东皇太一（中国神话第一个人格化的天帝形象）、少司命、大司命、山鬼。
  • 非人格化的至高神：昊天上帝/玉皇大帝、后土娘娘
  • 神兽：烛龙、应龙、白泽、麒麟、龙、凤凰、貔貅、四象（青龙-白虎-朱雀-玄武）

中国佛道神话

基本介绍

  • 印度佛教传入与道教产生后所形成的中国神话体系
  • 主要文献：各道教典籍、汉传佛教典籍、民间传说（如《西游记》、《封神榜》）
  • 相关文化背景：儒家文化、汉传佛教、道教
  • 佛教神系与道教神系更像是帝国官僚体系的神话投射，神话体系之间彼此交织
  • 多宗教并存

主要神系

  • 道教神系
    • 三清：元始天尊（盘古）、道德天尊（太上老君、老子）、灵宝天尊（太上道君）
    • 五方五老与六御天帝：道教的五方尊神、玉皇大帝与勾陈天皇大帝等天帝
    • 金母木公：西王母、东王公
    • 三官神阶：天官（赐福天官，紫微大帝）、地官（赦罪地官，清虚大帝）、水官（解厄水官，洞阴大帝）。
    • 星宿神阶：真武大帝、二十八星宿、北斗星宿等。

    • 地府阎罗神阶：十殿阎罗、地府下辖神祇。
  • 佛教神系
    • 三世佛神阶：横三世佛（中央释迦牟尼佛、东方药师佛、西方阿弥陀佛）、竖三世佛（往世佛燃灯佛、现世佛释迦牟尼佛、未来佛弥勒佛）。

    • 罗汉与金刚神阶：十八罗汉、四大金刚等神祇。
    • 婆罗门诸神神阶：天龙八部等印度佛教吸收印度婆罗门神祇后所形成的神祇形象。

    • 主流的行业守护神与家业守护神，如武圣关羽、妈祖、财神、门神、钟馗等。

everything搜索软件的使用和原理

发表于 2023-08-06 分类于 搞点课外知识
本文字数： 2.3k 阅读时长 ≈ 8 分钟

现在的面向大众的电脑硬盘容量越来越大，特别是最近长江存储颗粒的固态硬盘，2T的容量只要惊人的350块。但硬盘容量上去了，文件数量变多了，文件搜索也是一个大问题。除了使用规范化的文件夹命名、打开文件夹手动查找文件的方式以外，有时候还会使用Windows自带的文件搜索器，但这个搜索往往很耗时。所以这里就介绍一个可以充当Windows搜索功能的上位替代品的软件。

Everything

Everything是Voidtools制作组开发的文件搜索工具，官网描述为基于名称的实时文件和目录定位（Locate files and folders by name instantl），在2004年时创立项目，2008年推出了完善的1.0版本，目前最新的是2023年5月26更新的1.4.1.1024版本。

Everything（也称Everything Search Engine）软件堪称搜索界的神器，它是一个第三方的搜索软件，搜索速度特别快，而且CPU占用特别低，例如使用Windows系统自带搜索功能去搜索全盘，CPU、磁盘读写可能都会占用100%，且需要持续若干分钟，此时Everything只占用0.1%的CPU和磁盘读写，且只需持续个0.001秒，即使待搜索文件有几百万个，且对结果排序（例如按大小排序）都是毫秒级响应，使用时也不需要管理员权限。

使用方法

使用的时候也很简单，从官网的下载地址获取免安装的便携版，解压之后直接运行.exe文件，在首次或者隔一段时间后就会花个大概10秒钟进行硬盘数据的扫描（我的固态电脑大概366万的对象用了10秒不到的时间完成索引库的建立）。

如果是第一次运行，可能会弹出以下提示框，选择安装“Everything”服务即可。

接着在搜索框内输入待检索的内容（可以是文件/文件夹名字，也可以是.pdf等后缀扩展名），就能几乎是秒出排序结果了。

除了上面这种最近的搜索功能以外，还支持正则表达式、自定义的高级搜索等功能，不过这些我都没怎么用过。

快速搜索原理

Everything在第一次打开使用的时候，它会扫描整个硬盘，然后建立一个索引数据库（大小和电脑文件的数量有直接关系）。索引数据库建立之后，每次搜索文件，everything就在它的数据库里搜索，而不需要重新扫描整个硬盘，这也是everything和Windows系统自带搜索的本质区别。而everything之所有能快速地建立起这么小的索引数据库，主要得益于Windows对NTFS（New Technology File System，一种磁盘格式，除此之外还有FAT32等磁盘格式/文件系统）的支持。

MFT数据表

对于NTFS文件系统下的所有数据，都会有一个MFT数据表（Master File Table），该表记录了NTFS磁盘上所有文件的文件信息，且访问速度很快，Windows提供了各种API给开发者去直接调用并访问MFT数据表，不需要遍历文件系统就能获取当前卷/磁盘里的所有文件名称和路径。

USN Journal

除了MFT数据表以外，NTFS文件系统还有一个日志功能，即所有对文件系统有修改的操作都会被记录在USN Journal日志文件中，且该日志文件是实时、同步更新的。

Everything实现

Everything搜索引擎主要就是利用了NTFS文件系统中MFT和USN Journal的特性：

  1. 在程序启动时，扫描所有NTFS卷/磁盘的MTF表
  2. 将MTF表里面的文件名称以一种利于字符串检索的算法形式化存储在索引数据库中（在退出程序后就会看到有个大概一百多MB的Everything.db文件）
  3. 后台运行时，Everything会监控NTFS卷的USN Journal日志文件，实时更新该索引数据库
  4. 用户搜索文件时，Everything利用其字符串查找算法，在索引数据库里查找

从上面的实现过程可以看出，everything的搜索速度快、占用小，除了得益于其索引数据库生成算法和字符串查找算法以外，主要还是利用了MTF表的易于访问的特性。也正是因为这样，everything只能检索NTFS磁盘的文件，而不能检索其他如FAT32、exFAT等文件系统。

虽然everything比Windows自动的搜索功能快几个量级，但Windows也没有采取这种方法，可能主要原因还是因为文件系统的兼容性不足（仅局限于NTFS系统），且不支持搜索文件内容、附加属性。此外，微软自家的ActiveDirectory软件其实也有在使用MTF特性来快速检索文件。

LaTeX本地安装方法

发表于 2023-08-05 分类于 学术模块
本文字数： 1.1k 阅读时长 ≈ 4 分钟

LaTeX的官网是https://www.latex-project.org/，在另一篇博客《LaTeX与TeX的关系》也提到，LaTeX是使用TeX排版引擎生成PDF文件的一种语言，所以要使用LaTeX，就要有两个环境：（1）含有TeX引擎的TeX发行版，（2）一个LaTeX文本编辑器。

目前LaTeX可以在Linux、MacOS、Windows操作系统下部署并运行，也可以通过Overleaf、Papeeria、LaTeX base等在平台使用。对于在线使用而言，只需要一个浏览器进行在线的文本编辑就可以编译所需的PDF文件并下载，不需要额外安装TeX引擎并进行配置。但缺点也很明显，如果断网、网络不好，那用起来就很抓狂，所以这里我就只列LaTeX的本地（Windows）配置。

安装TeX排版引擎

对于Windows用户而言，可以无脑选择TeXLive，可以直接点开该TeXLive的官网链接，也可以在LaTeX官网中进行跳转。

下载并解压文件后，点击运行程序（一般是instal-tl-windows.bat），然后会在线下载、安装3G左右的内容，基本就是全自动完成的。

LaTeX编辑器-VSCode

LaTeX与Markdown

发表于 2023-08-04 分类于 学术模块
本文字数： 898 阅读时长 ≈ 3 分钟

Markdown和LaTeX是两种完全不同的标记语言，每一种都有其特定的用途和优势，前者相当于是轻量化版本的word，后者是用于生成规范化PDF文件的排版语言。

Markdown

Markdown是一种轻量级的标记语言，主要用于创建格式化的文本。

  • 它被设计为易读且易写，以便尽可能无障碍地进行文字表达
  • 有极简的语法，以纯文本方式书写，可以快速转化为HTML格式
  • 可以用来创建列表、链接、图片、代码块、标题、强调文本等
  • 大多数情况下在网络上编写和共享文档时使用，例如GitHub的README、博客和论坛上的说明文档

LaTeX

LaTeX是一种基于TeX的排版系统，被广泛用于科学、数学和技术文档的制作。

  • 提供了对复杂表格和数学公式的高级处理能力，以及对参考文献和索引的自动处理等功能
  • 语法更复杂，也有着更强的功能和灵活性

两者的主要差异

特性 Markdown LaTeX
易用性 语法相对简单，适合于编写简单的文档和笔记 提供了许多用于创建复杂文档的工具和选项，但需要更多的学习和理解
灵活性 主要关注内容本身，而不是样式 提供了对文档样式的深度控制，包括页眉、页脚、边距、脚注、参考文献等
公式表格处理 数学公式处理能力较弱，适合简单的表格 处理复杂数学公式和表格的最佳选择，其公式排版优美，符合学术要求
用途 更常用于编写网页、博客文章、演示文稿等 用于创建专业级别的科学和数学文档，如学术论文、技术报告、书籍等
输出格式 主要用于生成 HTML，但也可以通过一些工具转换为其他格式 可以输出到多种格式，如 PDF、HTML、RTF 等

总的来说，Markdown和LaTeX两者各有所长，选择使用哪一种主要取决于个人需求。如果需要编写一个简单的文档，那么Markdown 可能是一个更好的选择。如果需要创建一个包含复杂表格和数学公式的专业级别的文档，那么LaTeX可能是一个更好的选择。

LaTeX与TeX的关系

发表于 2023-08-04 分类于 学术模块
本文字数： 1.9k 阅读时长 ≈ 6 分钟

平常写论文有时候会接触到LaTeX和TeX两个概念，除此之外还会有pdfLaTeX、XeLaTeX、CTeX等等，这些名词之间大概是什么关系呢？

TeX排版引擎

TeX是一种宏语言，同时也是一种排版引擎，最初开发的时候就是为了能更方便地对文本进行排版，生成类似于PDF那样的规范化文件。TeX系统只有300多个元命令（primitive），十分短小精悍，但也很晦涩难读，一个简单的符号可能就需要多个命令来实现。

排版引擎

这里提及的排版引擎是底层的程序，其作用就是将输入语法进行解释，并把字排成行，把行排成页，涉及到断字、断行、分页等算法。最原始的排版引擎是TeX，此外还有其他排版引擎：

  • TeX：1978年由Donald Erwin Knuth开发，是后来大部分TeX相关的基础。它生成dvi文件，然后经由其他程序转换为pdf文件。
  • pdfTeX：TeX语言的一个实现，将TeX代码直接编译成PDF文件。
  • XeTeX：TeX语言的一个实现，支持Unicode编码和直接访问操作系统字体。
  • LuaTeX：TeX语言的一个完整的有扩展的实现。LuaTeX支持Unicode、系统字体和内嵌语言扩展，能直接输出PDF格式文件，也可以仍然输出 DVI 格式。

发行版

一个完整的TeX需要最基本的TeX引擎、格式支持、各种辅助宏语言包、一些转换程序、GUI、编辑器、文档查看器等等，将所有的程序、文件进行相应的组合，就形成了不同的发行版（一般都是用TeX Live）：

  • TeX Live⭐：支持Linux，Windows，Mac OS
  • MacTeX：只支持Mac
  • CTeX：只支持Windows，加入了中文的支持，可以很好地支持中文的宏包

LaTeX格式

格式

因为TeX本身很难用，所以后续就有LaTeX等其他基于TeX的语言，通过将最基本的TeX命令封装为各个更简便的宏命令，这些简写的合集就称为格式。LaTeX本身属于宏语言，也是一种格式。与TeX排版引擎一样，除了LaTeX以外，还有其他的格式（可以基于不同的排版引擎）：

  • LaTeX：更易于使用的宏集，最常见的一种格式。
  • Plain TeX：由Don Knuth提供的最小的宏集合。
  • ConTeXt：另一种常见的格式。

宏包

使用LaTeX的时候，还会有一些辅助文件来定义格式，这些辅助文件即为宏包（在LaTeX中叫做packages，在ConTeXt中叫做modules）。在LaTeX格式中，导言区的\usepackage的作用就是引入各种宏包。宏包其实也是一堆基本的TeX命令的集合，只是其不够全，所以称之为宏包而不是格式。

相关文件

LaTeX的最终目的是将文本按照特定的格式编译为PDF文件，同时为了简便使用，会把待编译的文本与排版格式分离开，因此就会有一个专门写文本的tex主文件，以及一系列定义排版格式的bib、sty等文件。

  • .tex文件：写文档内容的主文件，其余的辅助文件均在这里声明

  • .bib文件：写参考文献的文档，使用\bibligraphy方式导入参考文献时会用到

  • .bbl文件：.bib文件编译之后形成的文件，可以将里面的文本内容直接复制粘贴进.tex中

  • .sty文件：包文件，通常使用，定义各种排版风格

  • .cls文件：类文件，通过文档最前面的

  • .bst文件：定义参考文献格式的风格文件，一般各个期刊会有对应的文件（比如 ieeetran.bst）

LaTeX与TeX

LaTeX与TeX分别是格式与排版引擎，后者是底层运行的框架，前者则是基于TeX的几百个元命令所封装起来的语言，两者之间的关系类似于汇编语言与二进制语言。因此，在使用LaTeX时，本质上是在调用TeX的元命令，不过对于使用者来说则完全不需要考虑TeX的语法。

通胀与通缩-加息与降息

发表于 2023-08-03 分类于 搞点课外知识
本文字数： 1.8k 阅读时长 ≈ 6 分钟

最近总能看到新闻，说现在全球经济大环境不好，包括美国在内的大部分国家都在加息，而中国却在逆势降息。除了加息与降息，有时候还经常接触到通胀和通缩，为了更好地刷知乎、刷新闻，感觉还是得了解一下这四个经济学里面的概念。

通胀与通缩的定义

经济学现象 定义 根本原因 具体表现 影响
通货膨胀 货币贬值 货币流通量>货币需求量 物价上涨 刺激消费、扩大内需
通货紧缩 货币升值 货币流通量<货币需求量 物价下跌 消费下降、经济衰退

从上表可以看到，通胀或者通缩是由货币流通量以及货币需求量共同决定的，前者会受到各国中央银行的货币发行量（俗称放水）、外币兑换量、居民储蓄率等因素影响，后者则是由当前社会的商品生产量与价值决定的。理想情况下，货币的购买力与商品的产出供给匹配，那就相安无事。如果货币超发且其余因素不变，则会突然有大量的货币流通于市面上，但商品产值又没有发生变化，那么货币本身的价值就会降低。

通胀与通缩的评估：CPI

评估通胀或通缩的主要指标是CPI（社会消费者物价指数，Consumer Price Index），其高低表示物价的上涨程度，也表示通货膨胀的程度。一般在国家统计局等公开平台就能查到统计数据。

CPI指数 状态
<2% 通货紧缩
2%-3% 正常水平
3%-6% 通货膨胀
>6% 严重通货膨胀

通胀与通缩的影响

首先要明确的是，无论是通胀还是通缩，都是宏观经济运行的病态特征，有积极的一面，也会有对应的消极副作用，基本就是一种双刃剑。

通货膨胀

积极：

  • 投资行为增加。此时货币捏在手上就会贬值，而商品价值又在上涨，因而投资者更倾向于将手中的货币投入到社会生产中来避免资产贬值、实现资产增值。
  • 消费需求增加。最典型的例子就是居民预测商品房会涨价时，就会把所有积蓄都拿出来进行“消费”，甚至贷款去“消费”。
  • 扩大企业生产。当生产的商品价格在上涨时，企业会选择扩大生产规模以获取更大利润。

消极：

  • 居民资产缩水。此时货币购买力下降，居民储蓄能兑换的商品总额是在变少的，等价于资产在缩水。如果不能靠商品房等消费/投资行为实现资产增值，那最终就是资产越来越少。
  • 生产结构失调。资本是逐利的，企业会一直扩大生产规模，直到商品供应过剩，此时就可能会往通货紧缩演化。

通货紧缩

积极：

  • 减少经济泡沫。通过加剧市场竞争，在企业“你死我活”的竞争中促进企业改进产品和服务质量，从而调整经济结构。

消极：

  • 投资行为减少。此时货币本身在升值，手持货币啥也不干所获得的收益可能还高于具有风险的投资。
  • 消费水平下降。此时物价下降，消费者也预期商品价格会进一步下降，所以会持观望状，减少消费，这会进一步使得货币流通量降低，通缩加剧并进一步降低物价，形成恶性循环。
  • 失业人口增加。此时整个社会的商品需求处于萎缩期，生产商、服务商就会进行裁员，这种经济衰退自然会导致大量的失业人口。

加息与降息

不同于通货膨胀与通货紧缩这两个经济学现象，加息与降息是各国或地区的中央银行调整利息（存款利息和贷款利息）的行为，属于一种策略、手段。不过对于普通老百姓而言，通过观察各国地区的加息与降息状况，也能一部分反推出央行的意图，进而判断经济形势（然后和人吹牛逼）。

操作 定义 作用
加息 提高存款、贷款利率 抑制通货膨胀
降息 降低存款、贷款利率 刺激消费与投资

加息是一个国家或地区的中央银行提高利息的行为，会使商业银行对中央银行的借贷成本提高，进而迫使市场的利息也进行增加（不然大家都跑去市场里面借钱了）。加息最终可以减少货币流通量、抑制消费、控制通货膨胀、鼓励存款、减缓市场投机等等。

降息是与加息是逆向的操作，也是一种改变现金流动的金融方式。降息对于金融机构而言，放贷利率下降，融资的成本降低，因而就会促进投资行为。于普通个人而言，把资金存入银行的收益减少，因而会倾向于将资金从银行取出，将存款拿去投资或消费，最终结果是资金流动性增加。
