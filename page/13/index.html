如何确定研究领域大牛

发表于 2023-04-24 更新于 2023-06-09 分类于 学术模块
本文字数： 2.5k 阅读时长 ≈ 8 分钟

写在前面

又到周一了，这周五就提前开溜回家，迎来（翘班获得的）五一小长假，再加上最近看文献、调代码都比较乱，也没有指导纲领，所以现在完全没有科研动力捏（理直气壮.jpg）。所以，来搞点摸鱼、但又没那么摸鱼的事情，也就是学一学怎么确定某个研究领域的大牛都有哪些。有这个想法主要是因为昨晚学习磁共振的低秩重建时，发现了不少该领域的教授/团队，比如 UCB 的 Lustig（之前做过 CS-MRI 的）、USC 的 Haldar（居然还做过不少 SMS 的工作）、厦门大学的屈小波（整个实验室发的文章都很好，真羡慕啊），然后总结了一下，第一次感觉对这个领域有了更深的理解。或许要掌握一个领域（不管是学术的还是非学术的），除了要搞清具体的技术、物质层面上的概念和原理以外，人脉关系网也是一大重要的切入点呢！（或许也是看《显微镜下的大明》有感吧）

确定某个研究领域的大牛、实验团队，本质上就是一个学术搜索的事情。与论文搜索不同，确定研究领域的大牛需要更广泛、更综合的搜索才能得出结论，而前者一般只需要知道一个题目或者 DOI，即能在 scholar.google.com 或者 scihub 上定位并检索到。正是因为这种复杂特性，依靠自己多读文献或者多看综述这种方法注定不是最高效的方法，毕竟人工队师傅比不上机器队师傅嘛，因而掌握一些学术检索工具是非常重要的（而其他如公务员职场、金融市场下要如何获得对应情报，就得以后接触到的时候再学习了）。

这里给自己再强调一下，再好用的检索工具都只是一个情报搜索功能，只有自己认认真真看文献才能真正系统地建立起知识体系。

Web of Science

首先，WoS （www.webofscience.com）是一个对（至少是理科的）研究生而言特别重要的搜索引擎，其重要性甚至超过知网，因为其收录的国际期刊是更全的，而可预见的一段时间内，学术研究肯定还是以英文期刊为前沿的。要正常使用 WoS 的功能，要么使用带权限的账号（自己直接邮箱注册肯定没有的，这时候应该求助万能的淘宝），要么使用一般学校都会有的带权限账号，即通过中国高校联盟（CHINA CERNET Federation）登录（这里还有另一个叫 China CSTCloud ID-CAS 的选项，似乎是对应中国科技云通行证和身份认证联盟，应该和中国高校关系不大）。

然后选择 WoS 核心合集并进行文献主题的检索（目前为止的步骤仍与普通的论文检索一致）

AMiner 平台

AMiner（https://www.aminer.cn/）是清华唐杰教授开发的科技情报分析与挖掘平台，结合了 AI 的搜索和分析功能，自 2006 年上线，目前还为中国工程院、国家自然基金委、科技部等科研管理部门提供服务，除了常规的学术搜索功能以外，还能提供计算机顶级会议（好像对磁共振没啥用？）的 ddl 及链接入口、各领域必读论文、研究热点分析的功能，感觉有空的时候可以去瞄几眼。今天测试的时候（2023-0424）整个检索过程是可以免费使用的，不清楚以后会不会收费（我感觉是不会的，毕竟都这么多年了）。

回到本文的主题，在检索了研究领域主题之后选择专家，便能显示相关的作者清单，使用起来还是很方便的，不过结果可视化方面感觉没有 WoS 好看。

Conneted Papers

Conneted Papers（www.connectedpapers.com/）使用特别方便，文献之间关联的可视化效果做得特别好，它主要是通过某一篇具体的文章来搜索相关的领域（而不是直接检索一组研究领域关键词对应的文章），然后生成对应的文献关系图，进而间接地分析实验团队。该搜索平台在 2020 年 6 月上线，后续还在 2023 年 3 月更新了用户认证（具体发行历史可参考 https://connectedpapers.notion.site/Connected-Papers-Release-Notes-f758d434dfcc45e2b8ed3d0452739982），一般情况下需要注册使用，普通用户每个月只有 5 次免费机会，无限制账号的费用为 3 美元/月（非盈利或个人账号）或 10 美元/月（企业账号）。不过，可以在 Chrome 中通过 ctrl + shift + N 打开无痕窗口，即可不登录（也就无次数限制）使用，今天（2023-0424）亲测可用。

使用时在检索框内输入某一篇文章，然后生成统计分析图

随后就能查看拓扑结构中各节点的具体信息了，十分的好用

其他文献计量学软件

结构化矩阵

发表于 2023-04-23 更新于 2023-06-09 分类于 学术模块
本文字数： 1.2k 阅读时长 ≈ 4 分钟

结构化矩阵（Structured Matrix）是指可以用少于矩阵元素数量的参数去描述的矩阵，即如果一个 阶的矩阵只需要少于 个参数来描述，那这个矩阵就是结构化矩阵。通过“可以用更少的参量去描述该矩阵”这一特点，就能减少矩阵计算时的内存消耗，并利用快速的矩阵-向量乘法和梯度计算方法来极大地提高计算效率，当然，相应的代价便是近似时可能会引入一定偏差。

常见的结构化矩阵有 Hankel 矩阵、Toeplitz 矩阵：

（1）Toeplitz 矩阵（同一条对角线上的元素相等，2n-1 个独立元素） （2）循环矩阵（circulant matrix，特殊的 Toeplitz 矩阵） （3）Hankel 矩阵（同一条反对角线上元素相等）

Hexo插入图片

发表于 2023-04-21 更新于 2023-06-09 分类于 博客搭建

Hexo 的静态网页是通过 Markdown 格式的文本文件生成的，而在 Markdown 文件中插入图片的语法为 ![图片描述](图片路径)，其中图片路径可以分为以下三种。

  • 网络路径：以网址形式存储，十分方便，但源图片失效或网站限制（如微信公众号）时会无法加载图片，且访问速度较慢
  • 绝对路径：在计算机的绝对位置，不适用于博客的部署
  • 相对路径：相对于当前文件的路径

若要使用图片相对路径，则需要进行相应的配置，因为 Hexo 并不支持相对路径的引用。

根据习惯调整

上述存储方式实际上是与我自己的写作习惯冲突的：

  • 个人写作习惯：在 Typora 直接粘贴图片，图片默认保存路径为 ref_pic/imag.jpg 或 ./ref_pic/imag.jpg（后者需要将 Typora 升级到 1.4.3 之后的版本）
  • Hexo 图片索引规则：以 source 为根目录进行索引，且只能识别以斜杠 / 开头的相对路径。

接下来的改进需求是：

  • 核心需求：不改变以往的图片导入习惯，让 Hexo 能自动识别无 / 开头的图片相对路径
  • 次要需求：只维护一个图片保存文件夹，且该文件夹位置按照以往习惯保存在 .md 文件同目录下（即 \source\_post\ 下）

目前（2023-0423）只能满足核心需求：同时维护 \source\ref_pic 和 \source\_post\ref_pic 两个文件夹，前者用于 Hexo 识别，后者为了符合 Typora 使用习惯。此外引入 python 脚本，自动将 ](/ref_pic/ 或 ](/ref_pic 替换成 ](/ref_pic，需要每次 Hexo 编译前运行一次脚本。
