sci论文投稿流程

发表于 2023-05-23 更新于 2023-07-17 分类于 学术模块
本文字数： 2.5k 阅读时长 ≈ 8 分钟

论文撰写

完成了基本的数据、方法总结以及最核心的 Introduction 和 Abstract 后，论文的文本内容就已经基本完成了。这时候要先确定要投稿哪一家期刊，然后去它官网阅读作者投稿指南、下载论文模板（推荐 LaTex 写作，改格式要比 Word 快多了！）。投稿指南很重要，因为里面会讲许多论文的格式要求，比如不能超过多少页数（否则要加钱）、图片表格的字体与尺寸、论文结构要包含哪几个模块等等。

论文投稿

在找期刊的投稿指南时，也可以顺便找一下这个期刊的投稿入口，并注册期刊投稿账户。注册时一般都要绑定经过认证的 ORCID（开放研究贡献人员身份识别码，简称学者代码），这个差不多是学术领域的独一无二的身份证，认证的时候需要经过各高校的登录系统认证，要在学术圈里混的话，肯定就离不开这个 ORCID 了。

每个期刊的投稿系统所要填写的内容都可能不同，这里列出几点常见的：

  • 投稿状态：一般会有New Submissions”（新投稿）、“Revisions”（修改后再投）、“Completed”（接收后确认录用）三种选项。
  • 文档上传：有的可以打包压缩上传，大部分期刊是各文件分别上传，一般包括（1）正文 .tex 文件，（2）由 LaTex 生成的 .pdf 文件，（3）.pdf、.eps、.jpg 等各插图文件。各文件的命名参考投稿指南，IEEE Access 中是要求用论文标题命名 .tex 文件和由 LaTex 生成的 .pdf 文件，然后用 [作者名+index] 命名的正文插图文件，其中作者名为 5 个字符。
  • 创新性说明：一般会让你写一段本论文的创新性（150 字内），讲一讲贡献点在哪、如何与期刊的研究领域相符等。
  • 选择审稿人：普通实验室就不用管这些了，大概也只有经历大风大雨的实验室才需要避免让冤家仇敌做审稿人吧。
  • 选择专业领域：一般磁共振领域就是含这几个关键词的领域：Instrumentation and Measurement、Biomedical Engineering
  • 代码和数据开源声明：能选开源就尽量选开源，除非开源流程很复杂（比如代码和数据都要上传到指定服务器）。

等待审稿

完成所有材料上传和文本填写，并点击 submit 之后，剩下的就是等待审稿了。一般期刊投稿系统都能查阅当前的状态，从投稿指南里也能看到一共会有几个状态，而在这审稿期间又会涉及一些人员，因而下面的介绍也分为这几个部分（当然，主要还是参考一篇知乎文章的）。

  • 参与审稿的人：
  1. EIC ：Editor in Chief，主编，首先评判论文是否满足期刊要求，对稿件有最终决定权。
  2. ADM ：Administrator，协助主编工作。
  3. AE⭐：Associated Editor，副主编，会在审稿人（reviewers）意见的基础上对文章作个综合评价后，给主编一个recommendation。一般主编都会按照 AE 的意见写最终的 decision letter。（在学术会议里面 Program Chairs 会指定一批有建树的专家学者作为领域朱旭 Area Chair，即 AC，拥有一票否决权，地位基本与学术期刊里的 AE 一样）。
  4. TE ：Technical Editor，技术编辑，主要是熟悉文章领域的人对该文章进行宏观上的评价。
  5. SE ：Senior Editor，资深编委，应该跟主编差不多的身份吧。
  6. Reviewer⭐：审稿人，一般由 AE 委派（assign）三个审稿人对文章进行评审（俗称送外审），并将审稿意见返回给 AE。
  • 审稿状态：
  1. Awaiting Admin Processing ：一般 3-4 天后就会安排主编 EIC。
  2. Awaiting AE Assignment ：等待 AE 的指派。编辑部在选择/联系 AE，一般 1-3 天左右。
  3. Awaiting Reviewer Assignment ：等待指定审稿人。AE 在选择审稿人，等待审稿人回复是否同意审稿。一般在一周以内，看审稿人的回复速度。
  4. Awaiting Reviewer Scores：等待审稿人审稿意见。一般 AE 会要求审稿人三周内给审稿意见，如果审稿人觉得时间不够，可以写信给主编要求延长审稿期限。
  5. Awaiting AE Recommendation： 等待 AE 的推荐。一般要求 AE 三周内给结果。
  6. Awaiting TE Recommendation：等待技术编辑的推荐，一般技术编辑会根据审稿人的意见做决定.
  7. Awaiting SE Recommendation：等待资深编辑的推荐，一般是 1-2 天；
  8. Awaiting EIC Decision ：等待主编的决定，一般是 1-2 天。
  • 审稿结果：
  1. Accept：那只能恭喜你了！
  2. Accept after minor revision (without re-review)：不需要再送审。
  3. Major revision：大修，一般根据审稿人的意见进行修改录用的概率很大。
  4. Reconsideration after major revision：要再送审，即要再经过上述审稿流程。
  5. Reject and resubmit：论文现在状态不能接受，但可以修改后重新再投（其实差不多就是大修的意思，一般这时候主编都不会给论文一棒子打死，会说该文章有点新意，但是需要改进的地方还有很多，所以还有一个机会大修重投）。大修后的稿件要重新经过整个审稿流程。
  6. Reject ：一点希望都没有了，赶紧改投吧。

润色

参考的知乎文章中还推荐了两家润色机构：

  • Editage：国内公司，支持中文客服，有多种服务可选，价格相对较高，服务很好，回稿及时，提供发票；
  • Proofreading：国外公司，支持中文客服，有多种服务可选，性价比高，回稿及时，提供发票，不过需要通过英文邮件与编辑沟通（其实也没啥的）。

这里嵌套的两个 fftshift 操作是由于采集数据格式与傅里叶变换（FT）默认格式不同导致的。在 1D-FT 中，默认了输入数组 \([s_0,s_1,\cdots,s_{N-1}]\) 中的零频点（DC point，或者是时域上的原点）是第一个数据点 \(s_0\)。而在 MRI 采集的数据中，零频点是在输入数组的正中间。

两者默认的数据格式差异相当于是对输入数组进行了 \(N/2\) 的序号平移，因而为了满足 FT 的要求，需要对原始数据通过 fftshift 进行 \(N/2\) 的平移，并在 iFT 之后，再次通过 fftshift 将图像域的原点移到图像正中心，以符合看图习惯。

HTML-CSS-JS的关系

发表于 2023-05-23 更新于 2023-06-09 分类于 博客搭建
本文字数： 1.5k 阅读时长 ≈ 5 分钟

一般下，Web 前端有三剑客：HTML、CSS、JavaScript，这是三种不同的技术，实际应用时还需要相互搭配使用。

HTML

即超文本标记语言（Hyper Text Markup Language），是一种不严谨、简单的描述性语言。它用各种标签将页面中的元素组织起来，告诉浏览器如何显示内容。其中，“不严谨”是指，HTML 标签使用标准多，即使不加结尾也可以正常运行。

CSS

即层叠样式表（Cascading Style Sheets），是用来修饰 HTML 文件样式、控制网页外观的语言，负责字体、颜色、位置等外观的描述。其中，CSS 的“层叠”（cascading）表示“样式”（style）作用在同一个 HTML 元素的同一个属性时，具有优先级的层次结构，会依据层叠规则（权重）来处理冲突，优先选择应用权重更高的 CSS 选择器所指定的属性。

在 Hexo 博客中，要修改的 CSS 文件就是各主题文件里面的对应文件。具体到 NeXT 主题，就是 \themes\next\source\css\main.styl 文件。

JavaScript

一种嵌入到 HTML 页面中的脚本语言，简写为 JS，是用来做交互的。由浏览器一边解释一边执行，既可以运行在客户端（前端）也可以运行在服务器端（后端），用于扩展文档交互能力，使静态的 HTML 具有一定的动态功能（比如表单提交、动画特效、弹窗等）。注意，JavaScript 与 Java 完全没有关系，两者只是名称刚好相近（就像雷锋和雷锋塔）。

Hexo 中的 JS 文件主要就是 layout 文件夹中的各文件，具体到 NeXT 主题，就是 \themes\next\layout 里的各个 .swig 文件（swig 是一种 JS 模板引擎，见下文）。

与 Node.js 关系

JavaScript 是一种主要用于前端 Web 开发的脚本语言，而 Node.js 是一个后端用于运行 JavaScript 的环境，是适合初学者的流行且轻量级的 Web 框架，使得用户可以在任何浏览器上使用 JavaScript（包括前端、中间件、后端，以及 Linux、macOS、Windows）。

更具体一点，目前支持 JavaScript 的引擎（JS 引擎独立于它所在的浏览器，只不过各浏览器厂商倾向于用他们自己提出的 JS 引擎）主要有 Safari 的 JavaScript Core、FireFox 的 Spider Money、Google Chrome 的 V8 三种。而提供运行 JavaScript 环境的 Node.js，本质上用的就是 V8 引擎。

Swig 模板引擎

模板引擎本身是一个跨领域、跨平台的概念，在 Asp、PHP、C#、JavaScript 开发时都会用到模板引擎技术，其作用是将业务数据代码（内容）与界面逻辑代码（显示）分开，从而提高开发效率，降低项目维护难度。

具体到 Web 开发中的 JavaScript（JS）的模板引擎而言，JS 模板引擎是用于动态生成 HTML 的工具，负责将数据和模板结合。常见的 JS 模板引擎有很多，基本都可以直接通过 npm install 安装，以下就随便列几种：

  • EJS（Embedded JS）
  • Jade
  • Haml
  • artTemplate（腾讯的）
  • Swig🌟

其中，Hexo 默认使用 Swig 模板引擎，上手会简单一点。此外也可以另外安装插件了获得 EJS、Haml、Jade 的支持，Hexo 会根据模板文件的扩展名来决定所使用的目标引擎，例如：layout.ejs，layout.swig。
