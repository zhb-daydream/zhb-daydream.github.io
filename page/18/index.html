2023-0405日记

发表于 2023-04-06 更新于 2023-06-28 分类于 日记
本文字数： 314 阅读时长 ≈ 1 分钟

本来只是想弄个分类功能，然后发现默认的 landscape 主题好像不支持（不知道怎么设置），然后跑去下载了个 next 主题，结果生成的网页也只是代码。弄了一通发现单纯是 next 版本太久了（hexo g 的时候有 warning），更新到最新版本的 next 主题之后就能正常生成（还没有分类功能的）网页了。

现在才搞明白，原来 source 文件夹下面的几个文件夹，都是要有特定的名字的：

  • _posts：存放所有的文章（article）文件，对应的名字是可以任意的
  • categories：存放 index.md 文件，用于描述所有分类条目的名字
  • tages：存放 index.md 文件，用于描述所有标签条目的名字

Hexo 生成静态博客网站的大致流程：

  • 1 由作者按照 Markdown 格式撰写文章，存为 .md 文件
  • 2 Hexo 根据 [hexobin]\themes\source 文件夹中的 js、css、img 等静态资源将 [hexobin]\source\_posts\*.md 中的文章转换为对应的 html 网页，并存储在 [hexobin]\public 文件夹中，并将 index.html 作为入口访问博客
  • 3 Hexo 将 public 文件夹上传部署到对应的服务器中（如 github），使得可以进行外部访问
