HTML-CSS-JS的关系

发表于 2023-05-23 更新于 2023-06-09 分类于 博客搭建
本文字数： 1.5k 阅读时长 ≈ 5 分钟

一般下，Web 前端有三剑客：HTML、CSS、JavaScript，这是三种不同的技术，实际应用时还需要相互搭配使用。

HTML

即超文本标记语言（Hyper Text Markup Language），是一种不严谨、简单的描述性语言。它用各种标签将页面中的元素组织起来，告诉浏览器如何显示内容。其中，“不严谨”是指，HTML 标签使用标准多，即使不加结尾也可以正常运行。

CSS

即层叠样式表（Cascading Style Sheets），是用来修饰 HTML 文件样式、控制网页外观的语言，负责字体、颜色、位置等外观的描述。其中，CSS 的“层叠”（cascading）表示“样式”（style）作用在同一个 HTML 元素的同一个属性时，具有优先级的层次结构，会依据层叠规则（权重）来处理冲突，优先选择应用权重更高的 CSS 选择器所指定的属性。

在 Hexo 博客中，要修改的 CSS 文件就是各主题文件里面的对应文件。具体到 NeXT 主题，就是 \themes\next\source\css\main.styl 文件。

JavaScript

一种嵌入到 HTML 页面中的脚本语言，简写为 JS，是用来做交互的。由浏览器一边解释一边执行，既可以运行在客户端（前端）也可以运行在服务器端（后端），用于扩展文档交互能力，使静态的 HTML 具有一定的动态功能（比如表单提交、动画特效、弹窗等）。注意，JavaScript 与 Java 完全没有关系，两者只是名称刚好相近（就像雷锋和雷锋塔）。

Hexo 中的 JS 文件主要就是 layout 文件夹中的各文件，具体到 NeXT 主题，就是 \themes\next\layout 里的各个 .swig 文件（swig 是一种 JS 模板引擎，见下文）。

与 Node.js 关系

JavaScript 是一种主要用于前端 Web 开发的脚本语言，而 Node.js 是一个后端用于运行 JavaScript 的环境，是适合初学者的流行且轻量级的 Web 框架，使得用户可以在任何浏览器上使用 JavaScript（包括前端、中间件、后端，以及 Linux、macOS、Windows）。

更具体一点，目前支持 JavaScript 的引擎（JS 引擎独立于它所在的浏览器，只不过各浏览器厂商倾向于用他们自己提出的 JS 引擎）主要有 Safari 的 JavaScript Core、FireFox 的 Spider Money、Google Chrome 的 V8 三种。而提供运行 JavaScript 环境的 Node.js，本质上用的就是 V8 引擎。

Swig 模板引擎

模板引擎本身是一个跨领域、跨平台的概念，在 Asp、PHP、C#、JavaScript 开发时都会用到模板引擎技术，其作用是将业务数据代码（内容）与界面逻辑代码（显示）分开，从而提高开发效率，降低项目维护难度。

具体到 Web 开发中的 JavaScript（JS）的模板引擎而言，JS 模板引擎是用于动态生成 HTML 的工具，负责将数据和模板结合。常见的 JS 模板引擎有很多，基本都可以直接通过 npm install 安装，以下就随便列几种：

  • EJS（Embedded JS）
  • Jade
  • Haml
  • artTemplate（腾讯的）
  • Swig🌟

其中，Hexo 默认使用 Swig 模板引擎，上手会简单一点。此外也可以另外安装插件了获得 EJS、Haml、Jade 的支持，Hexo 会根据模板文件的扩展名来决定所使用的目标引擎，例如：layout.ejs，layout.swig。

SD与骨架插件ControlNet

发表于 2023-05-22 更新于 2023-06-09 分类于 搞点课外知识
本文字数： 2.1k 阅读时长 ≈ 7 分钟

Stable Diffusion（SD）提出之初还是主要靠 prompt（关键词）来进行绘画，同一组关键词往往会有不同的输出，最终就有点像是抽盲盒一样。但随着 AI 绘画模型的开源和插件化，个人也可以通过开源的代码库和插件训练并定制高质量的 AI 图像，此时关键词已经没那么重要了，比拼的是大模型以及各种模型、插件的综合运用。

ControlNet 就是其中一个很好用的 AI 绘画插件（Github 主页），可以通过骨骼绑定、控线来几乎完美地控制画面，在 2023 的 2 月中旬也一度成为最大热点。基本模式是输入一张参考图（甚至可以是简单的人物线条），然后插件根据参考图按一定模式预处理一张新图作为初始值，再由 AI 模型根据两幅图绘制出成品。

安装

插件

与汉化插件一样，可以在 SD-WebUI 的扩展界面从作者的 github 项目中加载可供下载的插件，搜索 sd-webui-controlnet 并安装（如果挂梯子也装得很慢，也可以通过 Install from URL 页面手动安装。插件地址是：https://github.com/Mikubill/sd-webui-controlnet）。

同时，还可以顺便安装 OpenPose Editor 插件，方便后续自己编辑自定义的姿势（pose）。

最后，再装一个能支持 ControlNet 的汉化插件（stable-diffusion-webui-chinese）。这次该插件无法从 SD-WebUI 作者的 github 项目中加载，因而要去“从网址安装”那里输入该插件的 github 地址（https://github.com/VinsonLaro/stable-diffusion-webui-chinese）然后安装。再去设置——用户界面——（最下方的）语言文件处，将语言文件从之前的 zh_CN 切换到该汉化插件（chinese-all-0512）。

ControlNet 应用模型

ControlNet 是需要专用模型的，否则无法使用相关功能引导画面。模型的下载路径为 IIIyasviel 的 huggingface 主页，其中 8 个常用的应用模型介绍如下（图文可参考一篇使用指南或作者说明文档），每个应用模型（文件后缀 .pth）为 1.45GB，有对应配套使用的 .yaml 文件作为预处理器文件（即 WebUI 中的预处理器选项），所有应用模型文件下载后的存放路径都为 \extensions\sd-webui-controlnet\models。尽管 civitai 里有 theally 提供了压缩版本的模型文件（后缀为 .safetensors，每个文件只要 689MB），但没有对应的 .yaml 文件，使用的时候可能会出现对应错误的情况，因而还是建议使用 IIIyasviel 版本的。此外还可以下载 T2I- Adapter 版本的 ControlNet 模型。

  • ⭐OpenPose：识别参考图像的姿势，提取出对应的骨骼姿势（甚至是手部骨骼），达到精准控制人体动作的效果。
  • ⭐Canny：识别参考图像的边缘，提取出线稿，再根据提示词生成同样构图的画面，也可以达成给线稿上色的效果。
  • HED：与 Canny 类似的边缘检测，但自由发挥程度更高，保留了更多参考图像的细节，绘制的人物明暗对比明显、轮廓感强
  • ⭐Scribble：提取参考图像的黑白涂鸦稿，自由发挥程度比 Canny、HED 更高
  • Mlsd：主要用于提取建筑的线条结构和几何形状，以构建出建筑外框
  • Seg：通过对参考图像进行语义分割，区分画面色块，适用于大场景的画风更改
  • Normal Map：提取参考图像中的 3D 物体的法线向量，使新图的光影效果与参考图像一致
  • Depth：提取参考图像的深度信息，生成具有同样深度结构的图或场景

预处理模型

第一次使用时都会自动下载预处理（preprocessor）所需要的模型并放到 \extensions\sd-webui-controlnet\annotator 的对应文件夹里，如果下载速度太慢也可以手动下载，参考秋叶的 b 站视频或者直接从lllyasviel 主页的 annotator 目录下载人体检测预处理模型（body_pose_model.pth 和 hand_pose_model.pth）至 /annotator/openpose、以及深度图模型（dpt_hybrid-midas-501f0c75.pt）至 /annotator/midas。

使用

上面配置正常的情况下，在 WebUI 界面就会出现 ControlNet 的选项。使用的时候，只是额外多了导入一张参考图片的步骤，并选择预处理器为 openpose，模型也为 openpose，其他的按照默认不用改动。

随后照常输入关键词，就能生成类似姿势的图像了。

注：ControlNet 依赖 xformers 算法框架及 Nvidia 显卡，因而使用 Mac 系统的用户使用效果可能只能有 70-80 分，而且速度慢，较难达到文档描述那种哇塞的效果。

StableDiffusion的使用

发表于 2023-05-22 更新于 2023-06-09 分类于 搞点课外知识
本文字数： 3.6k 阅读时长 ≈ 12 分钟

要使用 Stable Diffusion WebUI，只需要运行 stable-diffusion-webui/webui-user.bat 文件就可以了（里面的配置都已经在之前的安装博文里面设置好了），将其中显示的本地 URL 复制到任意浏览器中，进入一下的核心产图界面。

大模型选择

stable-diffusion 生成图像时主要依据大模型，这个倒没什么好说的，普通人一般用的就是著名的 ChilloutMix，无脑选就行。

提示词设置

这里只讲文生图的部分。要知道，基于 AIGC 的主流模型 Diffusion 提出来的 Stable Diffusion，是一种将文本转换为图片（text-to-image）的模型，因而提示词（prompt）差不多是仅次于大模型的最重要的参数。当然，由于 AI 所特有的黑匣子特性，产图过程是封闭的，就和魔法一样，因而提示词也常被称为“咒语”。而这些提示词/咒语/ tags 不知道怎么使用的话，优先参考 civitai 里面的 demo 参数，其次就是像 NovelAI Tag、魔咒百科词典这种热心大佬总结的网站。

除去直接套用现成的提示词以外，还是有几个小知识点可以掌握的，这里主要参考一篇青松pine的 CSDN 博文以及 WebUI 的 Wiki 说明。

语法

  • 英文输入，支持自然语言描述（不过还是推荐使用多个关键词来描述）
  • 英文逗号分隔（前后的空格、换行不影响）不同的关键词，而用 | 分隔的关键词表示同比例混合（若有多个关键词，则是包含了所有关键词的任意排列组合），一个例子如下：
  • 提示词权重有两种写法，第一种是 (提示词:权重数值)，权重数值范围是 [0.1,100]，默认为 1，低于 1 为减弱，反之加强。第二种写法是外面嵌套括号，每套一层 () 括号是增强 1.1 倍，每套一层 [] 括号是减弱 1.1 倍，例如 (((提示词))) 表示加强 1.1^3=1.331 倍。
  • 渐变：[关键词1:关键词2:数字] 表示先按照关键词 1 生成图像，等迭代 x 步数（x 小于 1 时为总迭代步数的前 x% 步）后开始往关键词 2 生成图像。

书写要点

  • 不是越长的魔咒（提示词）生成的图片越厉害，请尽量将关键词控制在75个（100个）以内。
  • 越关键的词，越往前放。
  • 相似的同类，放在一起。
  • 只写必要的关键词。

反向提示词(Negative Prompt)

AI 出的图是可能很丑的（比如三头六臂），需要用文字描述不希望在图像中出现的东西。反向提示词在 AI 中的大致流程是这样的：

  1. 对图片进行去噪处理，使其看起来更像你的提示词。
  2. 对图片进行去噪处理，使其看起来更像你的反向提示词（无条件条件）。
  3. 观察这两者之间的差异，并利用它来产生一组对噪声图片的改变
  4. 尝试将最终结果移向前者而远离后者
  5. 一个相对比较通用的负面提示词设置

Lora 模型

这个一般是在正向提示词（Prompt）里输入的。使用时与 Textual Inversion 一样，当做一个单独的关键词输入，不过使用时需要用尖括号，而且不能放到 Negative Prompt 里（Textual Inversion 则可以放在 Negative Prompt 和 Prompt 里），具体语法为：<lora:filename:multiplier>，其中 filename 是下载的本地 LoRA 文件的实际文件名，multiplier 一般是 [0,1] 间表示该 LoRA 权重的数值。使用 LoRA 时不支持用 | 连接进行混用，多个 LoRA 只能使用英文逗号连接。

其他绘图参数

采样方法 Sampling method

关于采样器方法的选择，可以参考别的大佬做出的详细总结：stable diffusion webui如何工作以及采样方法的对比。常用的采样器如下：

  • Euler a ：富有创造力，不同步数可以生产出不同的图片。 超过30~40步基本就没什么增益了。
  • Euler：最最常见基础的算法，最简单的，也是最快的。
  • DDIM：收敛快，一般20步就差不多了。
  • LMS：eular的延伸算法，相对更稳定一点，30步就比较稳定了
  • PLMS：再改进一点LMS
  • DPM2：DDIM的一种改进版，它的速度大约是 DDIM 的两倍

采样迭代步数 Sampling Steps

一般使用默认的 20 步就行。AI 绘画的大致原理是先生成噪声图片，然后再逐步往提示词方向调整图片，迭代步数越多，每次调整量就越小越精确。大部分采样器超过 50 步之后的意义都不大。下图是一组提示词在第 1 步到第 20 步的图像变化过程，可以看到差不多是 10 步之后结果就收敛了。

面部修复 Restore faces

对生成图片任务的面部进行修复，对真人效果好，很占显存。

可平铺 Tiling

生成可以往左右、上下连续拼接的图像

高清修复 Hires. fix

作用是将图片按照设置的放大倍率重新绘制图片内容。在勾选了 Hires.fix 选项后，会多出两行的选项

  • 放大算法：不知道选什么就选“ESRGAN_4x ”
  • 高清修复采样次数 Hires steps: 20 就行
  • 重绘幅度 Denoising：放大后细节的修改程度，从 0 到 1，数值越大，AI 随机程度越大，也就越偏离原图， 一般 0.5 - 0.7 就行
  • 放大倍率：即图像放大倍率，数值越大，对显卡能力要求越高

宽&高 Width & Height

单位像素。适当增加尺寸，AI 就会试图填充更多细节。较小的尺寸（小于 256 X 256）会让 AI 没地方发挥而导致图像质量下降；较大的尺寸（大于 1024 X 1024）也会让 AI 因乱发挥而导致图像质量下降。4GB 显存的极限大概是 1280 X 1280。常见的模型基本都是在 512 X 512、768 X 768 的基础上训练的，因而自己用的时候也可以用这种尺寸的配置，再用 Hires.fix 来生成高分辨率图像。

生成批次 Batch count / n_iter

用同样的参数循环跑的次数，每次产图时的随机种子都不一样，如果指定了随机种子，则相邻批次产图的种子数＋1。该选项是为了产生多个备选图像以供选择（毕竟 AI 不能保证每次出图都是好图）

每批数量 Batch size

每个批次下同时生成的图片数量，每次产图时的随机种子都不一样，每张的种子数＋1。与生成批次不同的是，增加这个值可以并行计算。例如，512 X 512 的图，使用 SD1.4 模型，Euler a 采样器，4G 显存可以并行 2 张图，8G 显存可以并行 8 张。

提示词相关性 CFG Scale

图像域提示词的匹配程度。一般取 [5,15] 的值，常用的值是 7，9，12，过高的值会让图像色彩过于饱和而降低了图像质量（此时，也可以通过增加采样迭代步数来抵消画质劣化）。

随机种子 Seed

AI 绘图的原理是先随机出一个噪声图片，再进行后续计算，该噪声图片作为初始输入就会影响最终的产图，而随机种子就是控制该随机过程的参数（计算机不存在真随机）。如果保持随机种子不变，则相同的模型、参数会大概率生成相同的图像，但要注意的是，即使采用完全一致的参数、模型，不同型号的显卡也可能会生成完全不同的图。

上图中，骰子图标表示快速输入 -1（随机值），绿色图标表示自动填写最近生成的图片的种子值。勾选右侧选项后会出现额外参数设置，其中差异随机种子 Variation seed 可以是 -1，也可以是固定值；差异强度 Variation strength 表示最终随机数是更偏向随机种子还是差异随机种子，值越大则最终的随机数越接近差异随机种子，值越小则越偏向随机种子。

几种git相关名词的基本概念

发表于 2023-05-19 更新于 2023-06-09 分类于 学术模块
本文字数： 1.3k 阅读时长 ≈ 4 分钟

进了互联网公司，核心产能就是代码，而管理多个人、多个项目的代码就基本是使用 Git。同时，搜索现成代码的时候，绕不开的一个平台就是 github，而有时候又会听人说用的是 gitlab 来管理代码，那么这些与 git 相关的概念都是什么意思呢？

Git

总的来说，Git 是一款免费、开源的分布式版本控制系统/工具。每个人的设备就是一个完整的版本库，支持离线工作，当需要多人协作、维护共同的版本库时，可以通过将各自的版本库 push 到统一的云端服务器/远程仓库进行管理。

与 Git 相对应的另一个概念就是 SVN（SubVersioN，版本控制系统），这是集中式版本控制系统。从名字上看可知，Git 与 SVN 的最主要区别就是 Git 是分布式的，SVN 是集中式的。对于 SVN 来说，版本库是集中放在中央服务器的，因而写代码时要一直联网工作，更适合与局域网下的协同工作。相比 SVN，Git 的概念更复杂、功能更丰富，因而 Git 出现后，慢慢地取代 SVN 成为主流。

与 python 一样，要在本地使用 Git 工具管理的话，也是需要去 Git 官网下载并安装的，之后就可以通过 Git Bash 调出控制台用命令行调用了（当然，还得配置一下），这也是 GitHub Desktop 等客户端的使用基础。

GitHub

就和 Sci-Hub 一样，xx-hub 里的 hub 表示的就是某个中心、社区的意思，而 GitHub 就是一个基于 Git 实现的在线代码平台，也是目前全球最大的代码托管平台，服务器都在美国（所以国内裸连速度会比较慢）。在 2008 年 4 月 10 日正式上线，许多大开源项目都在上面托管，可以提供免费的公共仓库、私有仓库，但涉及多成员协作的企业账号则需要收费。

Gitee（码云）

Gitee（也称码云，但这个名字真的不是马云谐音吗？） 是由 OSCHINA （深圳奥思网络科技有限公司，成立于 2008 年，是国内大型综合开源技术社区）推出的基于 Git 的代码托管平台，在 2016 年推出免费的企业版，相当于服务器在国内的 GitHub，区别是 Gitee 的企业账号不收钱。

GitLab

GitLab 是基于 Git 实现的在线代码仓库软件，有官方的服务器，可以像 GitHub 一样通过 网页 将代码存储在它的官方服务器上，此外还可以在企业、学校等内部网络上用 GitLab 自行搭建出基于 Git 的服务器（相当于是一种搭建 GitHub 私服的工具），具体的搭建、安装、使用可以参考 GitLab 官网的 下载页面 中的一些超链接。GitLab 最开始由 Ruby 语言写成，后面一些部分用 Go 语言重写，它核心的优势就是可以自己搭建封闭的代码管理平台，而不用使用 GitHub 的付费企业账号。

小结

名词 含义
Git 一种版本控制系统、一种工具
GitHub 基于 Git 实现的在线代码托管平台，企业版要收费
Gitee 基于 Git 实现的在线代码托管平台，企业版不收费
GitLab 基于 Git 实现的在线代码托管平台和软件，可以自己搭服务器
