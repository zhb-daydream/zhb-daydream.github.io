磁共振相关的python库

发表于 2023-05-05 更新于 2023-05-09 分类于 python
本文字数： 1.5k 阅读时长 ≈ 5 分钟

作为一个 python 小白，要做的第一件事就是找轮子。最近在弄 radial 的重建，在 MATLAB 上是已经找到了 NUFTT 的重建代码，调试好传入传出变量之后就无脑使用就行了，而 python 中用的比较多的 BART 却是只支持 Linux 的，感觉没必要专门为了一个网格化重建函数包就去专门弄个虚拟机吧？（虽然 Windows 的 WSL 好像也没那么复杂）所以还是得找一找相关的第三方库，顺带挖个坑，如果以后还用到其他库，就一并列进来好了。

NUFFT

参考 PyNUFFT-jyhmiinlin 的说明，有如下一些 python 中含有 nufft 库：

  • SigPy (Ong, F., and M. Lustig. "SigPy: a python package for high performance iterative reconstruction." )
  • gpuNUFFT (Knoll, Florian, et al. "gpuNUFFT-an open source GPU library for 3D regridding with direct Matlab interface." )
  • mrrt.nufft (mrrt.mri demos for the ISMRM 2020 Data Sampling Workshop in Sedona, AZ with raw cuda kernels)
  • pyNFFT (Keiner, J., Kunis, S., and Potts, D. ''Using NFFT 3 - a software library for various non-equispaced fast Fourier transforms'' )
  • python-NUFFT (Dan Foreman-Mackey, Thomas Arildsen, and Marc T. Henry de Frahan, but the code that actually does the work is from the Greengard lab at NYU)
  • finufft (Barnett, Alexander H., Jeremy Magland, and Ludvig af Klinteberg. "A Parallel Nonuniform Fast Fourier Transform Library Based on an “Exponential of Semicircle" Kernel." )
  • torchkbnufft (M. J. Muckley, R. Stern, T. Murrell, F. Knoll, TorchKbNufft: A High-Level, Hardware-Agnostic Non-Uniform Fast Fourier Transform)
  • tfkbnufft (adapt torchkbnufft for TensorFlow)
  • TFNUFFT (adapt the min-max interpolator in PyNUFFT for tensorflow)
  • tensorflow-nufft

其他

  • Bart（UCB 开发的 MRI 重建库，包含了 SMS、pMRI 等一堆前沿重建方法，缺点是暂时只能在 Linux 下运行）
  • FSL（英国牛津大学开发的专门用于 fMRI 的库，包含了图像配准的一些预处理方法，缺点也是只能在 Linux 下运行）

查找 python 第三方库

Python 的一大特性在于有大量的用户群基数以及庞大的社区，也有很多善心大发的大佬免费将对应的代码整理为完善的第三方库以供使用，因而使用 python 时的一个重要特性就是 不要重复造轮子，有现成的函数库就用现成的，若是真的不好用、有定制需求才开始自己写（当然，如果你是大佬能随便手撕代码的话那也无所谓了）。

磁共振领域里，有挺多的第三方库，比如 Bart（UCB 开发的 MRI 重建库，包含了 SMS、pMRI 等一堆前沿重建方法，缺点是暂时只能在 Linux 下运行）、FSL（英国牛津大学开发的专门用于 fMRI 的库，包含了图像配准的一些预处理方法，缺点也是只能在 Linux 下运行）等等。更多的我也列举不出来了，不然我也不会想去搜其他第三方库，也就不会有这篇博文了。

虽然网络里都是宝，但也要学会如何去找 python 的第三方库，除了直接问师兄师姐这种吃人嘴短的（呜呜，有人带、有现成的代码库还是很羡慕的好吧）方法以外，大概有这么以下三种途径：

1 搜索引擎

在百度、必应、谷歌上去搜信息是最直接也最简单的方法，核心就是要有正确的关键词。首先要明确自己的需求，打算用 Python 来干什么，力求简短明了，比如需要使用 NUFFT 的操作，则搜索以下关键词（用空格连接），一般情况下就会出现很多博客或知乎回答推荐的第三方库，再看看是否适合自己，进而再去搜索该库所对应的主页（一般大佬都会专门弄个域名和在线文档以供教学）。

但这种方法有时候很难精准搜索，而且很多冷门库没什么内容，搜索引擎自然无法抓取到你想要的库。

2 PyPi

PyPi 是 Python 的包索引平台，全称 Python Package Index，所有的第三方库都在里面，估计有上万个。在主页处就有搜索功能，可以搜索你想要的库，也可以通过功能区索引查询相关的库。

点击 browse projects 会进入检索页面，里面有 topic 检索，也就是根据库的主题来查找（比如，想查找关于金融方面的 Python 第三方库时，可以先找到 Financial，然后点击，就会出现所有金融相关的库，同时也可以进行组合搜索，比如金融+会计）。选好一个库后，点进去会看到该库的详细介绍和简单的教程。

总的来说，PyPi 是查询 Python 第三方库最好的平台，因为最全、最新、最详细，但缺点就是同一类目库太多了，新手很难去甄别哪个更适合用。

3 Github

万能的 Github 是全球最大的代码托管网站，里面聚集了全球的码农和最优秀的代码项目，很多 Python 库的代码都托管在 Github 里，比如 pandas、matplotlib、tensorflow 等，同时 Github 里也有很多Python爱好者开发的资源项目，包括很多库的应用以及整合。

比如有个项目叫 awesome-python，就分门别类整理了几百个热门库及其简介，涉及Web 框架、网络爬虫、网络内容提取、模板引擎、数据库、数据可视化、图片处理、文本处理、自然语言处理、机器学习、日志、代码分析等十几个场景。还有个有趣的项目是 HelloGithub，作者每周一期收集了 Github 上各种有趣的项目，其中就包括Python的各种库。
