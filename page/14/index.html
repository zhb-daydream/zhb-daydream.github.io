几种git相关名词的基本概念

发表于 2023-05-19 更新于 2023-06-09 分类于 学术模块
本文字数： 1.3k 阅读时长 ≈ 4 分钟

进了互联网公司，核心产能就是代码，而管理多个人、多个项目的代码就基本是使用 Git。同时，搜索现成代码的时候，绕不开的一个平台就是 github，而有时候又会听人说用的是 gitlab 来管理代码，那么这些与 git 相关的概念都是什么意思呢？

Git

总的来说，Git 是一款免费、开源的分布式版本控制系统/工具。每个人的设备就是一个完整的版本库，支持离线工作，当需要多人协作、维护共同的版本库时，可以通过将各自的版本库 push 到统一的云端服务器/远程仓库进行管理。

与 Git 相对应的另一个概念就是 SVN（SubVersioN，版本控制系统），这是集中式版本控制系统。从名字上看可知，Git 与 SVN 的最主要区别就是 Git 是分布式的，SVN 是集中式的。对于 SVN 来说，版本库是集中放在中央服务器的，因而写代码时要一直联网工作，更适合与局域网下的协同工作。相比 SVN，Git 的概念更复杂、功能更丰富，因而 Git 出现后，慢慢地取代 SVN 成为主流。

与 python 一样，要在本地使用 Git 工具管理的话，也是需要去 Git 官网下载并安装的，之后就可以通过 Git Bash 调出控制台用命令行调用了（当然，还得配置一下），这也是 GitHub Desktop 等客户端的使用基础。

GitHub

就和 Sci-Hub 一样，xx-hub 里的 hub 表示的就是某个中心、社区的意思，而 GitHub 就是一个基于 Git 实现的在线代码平台，也是目前全球最大的代码托管平台，服务器都在美国（所以国内裸连速度会比较慢）。在 2008 年 4 月 10 日正式上线，许多大开源项目都在上面托管，可以提供免费的公共仓库、私有仓库，但涉及多成员协作的企业账号则需要收费。

Gitee（码云）

Gitee（也称码云，但这个名字真的不是马云谐音吗？） 是由 OSCHINA （深圳奥思网络科技有限公司，成立于 2008 年，是国内大型综合开源技术社区）推出的基于 Git 的代码托管平台，在 2016 年推出免费的企业版，相当于服务器在国内的 GitHub，区别是 Gitee 的企业账号不收钱。

GitLab

GitLab 是基于 Git 实现的在线代码仓库软件，有官方的服务器，可以像 GitHub 一样通过 网页 将代码存储在它的官方服务器上，此外还可以在企业、学校等内部网络上用 GitLab 自行搭建出基于 Git 的服务器（相当于是一种搭建 GitHub 私服的工具），具体的搭建、安装、使用可以参考 GitLab 官网的 下载页面 中的一些超链接。GitLab 最开始由 Ruby 语言写成，后面一些部分用 Go 语言重写，它核心的优势就是可以自己搭建封闭的代码管理平台，而不用使用 GitHub 的付费企业账号。

小结

名词 含义
Git 一种版本控制系统、一种工具
GitHub 基于 Git 实现的在线代码托管平台，企业版要收费
Gitee 基于 Git 实现的在线代码托管平台，企业版不收费
GitLab 基于 Git 实现的在线代码托管平台和软件，可以自己搭服务器

StableDiffusion的几种模型

发表于 2023-05-18 更新于 2023-06-09 分类于 搞点课外知识
本文字数： 2.1k 阅读时长 ≈ 7 分钟

为了避免一篇博文太长，想了想还是决定把 Stable Diffusion 涉及的几种常见模型单独列出来讲。在 SD 的安装教程中只提到 SD 提供的基础 checkpoint（如 sd-v1.4.ckpt），但实际还有一大堆大神弄出来的各种其他模型以及对应的微调工具，这可不得不了解一下。

Stable Diffusion 框架会用到几类模型，基本都可以从万能的 c 站（www.civitai.com）下载（从网页右上角的 filter 中也可以看到模型的分类），这几种常见模型类型的区别可以参考 一篇 CSDN 博文 或者 civitai 的说明。访问 c 站的时候可以右上角登录自己的 google 账号，然后记得将 Browsing Mode 调成 Safe，不然就会有一大堆 NSFW（Not Safe For Work）的内容出现，谨慎呀！

checkpoint（主模型）🌟

在深度学习中，checkpoint 是一个用于描述每次训练完后保存模型参数、权重操作的术语，就像在游戏中的存档点，而在 AI 绘图中，更像是一个画师（后续的其他类型模型则属于调整和优化主模型的工具）。在前文的 WebUI 安装中提到的 "No checkpoints found" 错误即是缺失主模型文件。checkpoint 也就是 StableDiffusion 里所谓的大模型，或者说基础模型，比如 SD 官方自带的 sd-v1.5.ckpt（我 2 月份下载的时候还是 sd-v1.4.ckpt）。这类模型都是由 dreambooth 技术，经过大量数据训练得到的，可以用于生成各类常见元素的图片，算是一个全能工具。因此不可缺少，至少有一个，不然 SD 不能启动。

SD 官方的 checkpoint 可以从 这里 下载（我也不知道这个是不是官方链接），而之前火出圈的真人绘图则主要是基于 ChilloutMix 模型，基本在 civitai 里都能找到。下载好 checkpoint 文件后，存放位置是 /models/Stable-diffusion 文件夹，文件的后缀一般为 .ckpt 或 .safetensions，这种文件一般为 4GB 左右的大小。

Embedding（文本转换，Textual Inversions）

Embedding 是指将自然语言文本（如句子或段落）转换为计算机可以理解的数值向量表示形式的过程，这种向量表示法通常称为嵌入（embedding），可以在许多自然语言处理（NLP）任务中使用，例如语言模型、情感分析、问答系统等。在Stable Diffusion中，用户可以使用内置的嵌入模型或创建自定义的嵌入模型来生成嵌入，但最终效果特别依赖 checkpoint，一定要看清适用于什么 checkpoint。下载好 embedding 文件后，存放位置是 /embeddings 文件夹，文件的后缀一般为 .pt，这种文件一般在 1MB 以内。

LoRA🌟

LoRA 全称为 Low-Rank Adaptation of Large Language Models（大语言模型的低阶适应），最初是由微软为了解决大语言模型的微调微调而开发出来的一项技术。在 Stable Diffusion 中使用时相当于绘画风格滤镜，对我这种小白用户来说是最重要的一类模型，因为往往只需要使用不同的 LoRA 模型就能满足相应绘画需求了。下载好 LoRA 文件后，存放位置是 /models/Lora 文件夹，文件的后缀一般为 .pt 或 .safetensions，这种文件一般为 144MB 左右的大小。

Hypernetwork

在 Stable Diffusion 中，Hypernetwork 是一种用于训练大型神经网络的技术，是一种生成网络的网络，可以通过它来生成其他网络的权重。相比 LoRA，多了一些步骤，更适合风景绘图，而不适合特定具象的物体，使用的人也比较少。下载好 Hypernetwork 文件后，存放位置是 /models/hypernetworks 文件夹，文件的后缀一般为 .pt，一般为 100MB 左右的大小。

VAE

在Stable Diffusion中，VAE 是另一种用于训练大型神经网络的技术，与 Hypernetwork 一样也是一种生成网络的网络。其功能是增强模型的色彩、光照等表现效果，相当于给模型加了美颜滤镜（如果模型生成的图发灰，可以检查一下 Vae 文件是否勾选），不过大都会集成在主模型 checkpoint 中，一般也不会单独列出。下载好 Vae 文件后，存放位置是 /models/VAE 文件夹。

小结

主要还得看 LoRA 模型，其次就是再关注一下 checkpoint 和 Embedding，剩下的模型都是非热门的，只想浅尝辄止的话就可以不用管了（大伙会用下载量投票的）。还有就是要注意一下本地文件的存放的位置，不要放错地方了哦。

害，前段时间 AI 画图特别火，为了不让自己落后于时代，我也跟着教程去下了个 Stable-Diffusion，然后就丢文件夹里吃灰了😓。本来想趁还记得，赶紧写个博客的，结果 2023-0224 安装好，一直拖到接近 3 个月之后才记录，太懒惰了。好像 4 月份又出了个上手更快一点的 midjourney，看来到时候也得了解一下了。（再扩展地想一想，最近 NLP 的 AI 也是特别火，从最开始 OpenAI 的 ChatGPT，到后面必应的 New Bing、百度的文心一言、谷歌的 Google bard，是不是也得了解一下呀。）

本文的主题是 Stable Diffusion AI 画图的安装，当初学的时候主要就是参考 b 站 爱吃鱼的番茄 的视频，这种热门技术的安装教程应该也一搜一大堆的。基于 diffusion 扩散模型（DDPM）的方法在 2020 年就被提出来了，是 AIGC 的主流模型，主要用来生成图片，然后在去年（2020 年）又提出了 stable diffusion，是一种将文本转换为图片（text-to-image）的模型，而真正让这个 AI 绘画火出圈的应当是后续的真人模型（lsp 的胜利），训练得到的不同风格模型会产生不同的图片。因而，安装过程也可以分为类似的几个模块讲。

1 python3.10

基础代码平台是 python 3.10.6（更早版本的 python 不支持 torch 库，更新的版本可能不稳定，所以还是直接 3.10.6 吧），直接从 python 官网下载就行，在安装的时候要注意添加环境变量。如果碰见一个电脑上需要装多个 python 版本的话，可以用 Anaconda 来管理？我这部分也没太搞懂，目前我电脑就是装了有一个 py310 的版本，但平常用的还是 py3.8的。

2 Stable-Diffusion-WebUI

首先要明确的是，Stable Diffusion 是类似于一个引擎、框架，现在大家用的基本是一个 github 大佬开发的 WebUI 版本，其特性就是可以本地部署，用网页界面交互、用自己的 GPU 生成图片，而不需要去其他的网页（比如 DreamStudio ）中注册账号受限使用。安装 Stable Diffusion WebUI 时，可以去它的 github 主页 手动下载，或者自己进行 git clone，或者找别人已经打包好的安装包。在本地安装的时候，记得留个相对较大的地方，虽然 Stable Diffusion WebUI 作者在 github 上的源码一共也就 2MB 不到，但后续还需要往这个文件夹里放各种模型以及生成的图片，我目前该文件夹就已经用了 14.3 GB 了。

4 运行并配置中文

再次运行 stable-diffusion-webui/webui-user.bat 文件，顺利的话就会出现以下界面啦，将显示的本地 URL 复制到任意浏览器中，就算是完成安装啦！

但此时还有个重要的问题，那就是默认下全！是！英！文！虽说英文对于过了四六级的大学生来说也不是不能看，但这么多专业名词，中文都不一定懂，还看英文那不是折寿嘛，所以汉化完才算是安装完毕能交付的。

要安装中文插件，也不需要额外找，就用 Stable Diffusion WebUI 大佬提供的就行。打开网页，然后按下图顺序先后选择 Extension—— Available —— 取消勾选 localization（即取消隐藏各语言的汉化包）——点击 Load from。

接着在该页面的下方就会有一堆的 Extension，直接 Ctrl+F 找到 zh_CN Localization 的插件并安装（这个是简体中文的，还有另一个是 zh_TW Localization 的是繁体中文，但这个 TW 又是什么鬼？）

安装好之后，可以先去 Extension —— Installed 那里看看有没有勾上汉化包，然后再点 Apply and restart UI。

如果还没生效的话，就去 Settings —— User interface 的最下方找 Localization (requires restart) 选项（也可以在左侧点 Show all pages 然后全文搜索），最后回到上方点个 Apply settings 然后 Reload UI 就行啦！我这里就懒得放图污染文章了。

背景补充

emoji 表情是目前网络用的比较多的表情符号（划重点：本质是符号，不是图片），可以在任何支持文本输入的地方使用，目前由统一码联盟（Unicode Consortium）进行规范和管理，即每一个 emoji 在 Unicode 标准里都有其对应的码位，因而也保证了它可以在不同设备和平台上通用（当然，同一个 emoji 在不同设备/平台上可能会显示不一样）。Emoji14.0（2021-0914 推出）里就已包含 3000 多个符号，后续还会不断更新其他版本的 emoji。
